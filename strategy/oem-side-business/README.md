# コンサルOS：副業OEM販売戦略

このコンサルOS（Claude Code ベースのコンサル業務支援システム）を、
他社にOEM販売＋ランニング品質改善サポートとして提供する事業の戦略ドキュメント。

判断モード：佐藤裕介フレーム（キャッシュフロー直視・幻想排除・撤退基準先決め）
事業ポジション：副業路線（本業を阻害しない範囲で、少数・高単価・低工数）

---

## 戦略サマリー（一枚で読む用）

| 項目 | 結論 |
|---|---|
| 事業位置づけ | 副業（本業のキャッシュフロー補完。スケール狙わない） |
| 顧客数上限 | 同時進行 2〜3社まで。それ以上は受けない |
| 価格レンジ | 初期導入 300〜500万 / 月額 50〜100万 |
| 工数上限 | 1社あたり月20時間以内。超過分は追加課金 |
| カスタマイズ | 原則しない。するなら別料金（人月単価で見積） |
| 契約期間 | 1年単位。自動更新あり、解約は60日前通告 |
| 撤退基準 | 月工数20時間を3ヶ月連続で超過したら値上げか解約 |
| 想定LTV | 24〜36ヶ月（剥がされる前提でOK） |

## なぜ「副業路線」なのか（判断ログ）

> 佐藤裕介フレームでの判断結論：今のままならNO、条件付きでYES

- OEM販売は本来「自社販売の機会損失」になりがち
- ランニング品質改善は見た目SaaSっぽいが、中身は人月。スケール曲線が立たない
- OEM先は売れたら自社化したがるので、3年で剥がされる前提のビジネス
- → 本業として張るには弱い。だがキャッシュフロー副業としてなら成立する

副業として成立させる前提：

1. 顧客は2〜3社まで（本業の時間配分を死守）
2. 高単価・低工数の構造を契約レベルで担保
3. カスタマイズは原則しない
4. ナレッジ資産（テンプレ・ベンチマーク）で殴る、人月で殴らない
5. 3年で剥がされてもOKという覚悟（LTV最大化に執着しない）

詳細は [`decision-log.md`](./decision-log.md) 参照。

## 使い方（オペレーション）

1. 引き合いが来た → `03-ideal-partner-profile.md` で受ける／断るを判断
2. 受ける場合 → `01-service-definition.md` を提案資料の土台に流用
3. 見積依頼 → `02-pricing-and-contract.md` の料金表をそのまま提示
4. 契約 → `02-pricing-and-contract.md` の契約条項を契約書ドラフトに転記
5. 運用中に判断ブレた時 → `decision-log.md` の撤退基準で再評価

## このフォルダの位置づけ

これは「コンサルOSを使ってコンサルOS自身の事業戦略を書いた」自己言及的ドキュメント。
副業として実際にOEM案件を受けるたびに、本ドキュメントを更新していく。
