# NotebookLM の本質：RAG型の知識検索ツール

| NotebookLM が得意なこと | 関根さんが求めること |
|---|---|
| 文書から関連情報を**検索・引用** | 関根さんの**思考・判断・規律**を再現 |
| 「何が書いてあるか」を返す | 「関根さんならどう判断するか」を返す |

### 結論

NotebookLM は<span class="highlight">「辞書」</span>、<br>関根さんが欲しいのは<span class="accent">「判断装置 × 専門家チーム × 品質ガード」</span>。

→ 設計思想が違うので、ソースを足しても辞書のまま。

# コンサルOS：規律ある専門家チームを再現する事業OS

### 6 部門・30 エージェント

| 部門 | エージェント数 | 役割 |
|---|:---:|---|
| コンサル | 7 | 戦略・提案・KPI・法務 |
| サービス開発 | 4 | 実装・AI・インフラ |
| プロダクト | 2 | PM・VOC |
| クリエイティブ | 6 | デザイン・コンテンツ・グロース |
| グローバル | 3 | GTM・現地・翻訳 |
| マーケ調査 | 8 | 広告・SEO・分析・SNS・PR |

→ **経営判断から実装・マーケまで一気通貫**で動かせる構造

---

# 19 スキル＋ハードルール＋継続改善

### 19 スキル（プレイブック）
- コンサル / クリエイティブ / エンジニア各プレイブック
- **業界プレイブック5業界**（広告代理店 / D2C / 金融 / 製造 / SaaS）
- **反証モード**（トリプルチェック品質ガード）
- プロンプトテンプレ30本・ブランド／デザインガイド

### ハードルール組込み（CLAUDE.md）
- **PL試算なし提案を禁止**（粗利インパクト必須）
- **出典なし数値断言を禁止**（FACT/INFERENCE/SPECULATION 3 ラベル）
- **反証チェック**（トリプルチェック）必須
- **属人営業推奨を禁止**（佐藤メソッド：構造で売る）

### 継続改善基盤
- **evolution-log** — 規律違反を学習記録
- SessionStart hook + GitHub Actions 品質ゲート
