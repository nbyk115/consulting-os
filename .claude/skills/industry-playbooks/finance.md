# 金融業界プレイブック

> ⚠️ **ハルシネーション禁則 + 法的助言不可**: 本プレイブックは**法的助言ではない**。金融業界は規制が極めて厳しく、提案・分析時は必ず**法務・コンプライアンス部門 + 弁護士のレビュー**を経ること。数値・規制内容は**必ず一次ソース（金融庁・各業界団体）で最新版を確認**。

## 1. 業界構造

### 業種分類
- **銀行**: メガバンク / 地方銀行 / 信用金庫 / ネット銀行
- **証券**: 大手証券 / ネット証券 / 投信運用
- **保険**: 生命保険 / 損害保険 / 共済
- **資産運用**: 投資信託 / プライベートバンキング / VC / PE
- **決済・キャッシュレス**: クレジットカード / QR コード決済 / 電子マネー
- **暗号資産・Web3**: 暗号資産交換業者 / NFT
- **FinTech**: ネオバンク / 与信 SaaS / 会計 SaaS / マネーフォワード等

### 業界構造の特徴
- **規制業種**: 金融庁の許認可が必須。ほぼすべての業務に規制
- **顧客資産の受託**: 信託責任・受託者責任が重い
- **システム障害許容度ゼロ**: ATM・取引システム停止は社会問題化
- **AML/CFT**: マネロン・テロ資金供与防止が継続課題
- **DX 後発**: 規制とレガシーシステムで DX が他業界より遅れる傾向
- **FinTech との協業 / 競合**: 銀行 API 開放（電子決済等代行業）、BaaS 化進行

## 2. 業界固有 KPI

> ⚠️ 銀行・証券・保険で KPI 体系が大きく異なる。**断定数値の引用禁止**。各社 IR / 業界白書で最新値確認。

### 銀行
- **ROE / ROA**: 自己資本利益率・総資産利益率
- **NIM**（Net Interest Margin / 預貸利ざや）
- **OHR**（Overhead Ratio）: 経費率
- **不良債権比率（NPL Ratio）**
- **自己資本比率**: バーゼル III 規制（最低水準あり）
- **預貸率 / 預証率**: 資金運用バランス

### 証券
- **AUM**（Assets Under Management）: 預かり資産
- **取引高 / 手数料収入**
- **アクティブ口座数 / 新規口座開設**
- **ROE**

### 保険
- **コンバインドレシオ**（損保）: 損害率 + 経費率
- **ソルベンシーマージン比率**: 健全性指標
- **新契約価値（NBV）**（生保）
- **解約率**

### FinTech / 決済
- **MAU / DAU**
- **決済取扱高（GMV / Volume）**
- **アクティブユーザー数**
- **テイクレート**（手数料率）
- **CAC / LTV**

## 3. 業界規制・コンプライアンス（極めて重要）

> ⚠️ 金融規制違反は**業務停止命令・行政処分・刑事罰**の対象。本プレイブックの内容は**最新の金融庁ガイドラインで必ず確認**。

### 共通規制
- **金融商品取引法（金商法）**: 証券業・投資助言・ファンド運用全般
- **銀行法**: 銀行業
- **保険業法**: 保険業
- **資金決済法**: 暗号資産・前払式支払手段・資金移動業
- **割賦販売法**: クレジットカード・包括信用購入あっせん
- **貸金業法**: 消費者金融・カードローン

### マネロン・テロ資金（AML/CFT）
- **犯罪収益移転防止法**: 本人確認・疑わしい取引届出
- **FATF 勧告**: 国際基準、日本は審査対象
- **金融庁 AML/CFT ガイドライン**

### 顧客本位の業務運営（Fiduciary Duty）
- **金融庁「顧客本位の業務運営に関する原則」**
- **FD（Fiduciary Duty）原則**

### システム・サイバーセキュリティ
- **FISC 安全対策基準**: 金融機関の情報システム安全対策
- **金融庁 サイバーセキュリティガイドライン**
- **データ保護**: 個人情報保護法 + 金融分野の特則

### 広告・募集
- **金商法 適合性原則**: 顧客属性に応じた商品提案
- **断定的判断の提供禁止**: 「絶対儲かる」等は禁止
- **誇大広告禁止**: 不確実な事項を確実と誤認させる表示禁止

### FinTech 特有
- **電子決済等代行業者登録**: 銀行 API 接続事業者
- **資金移動業ライセンス**: 送金事業者
- **暗号資産交換業者登録**: 暗号資産取引所
- **改正資金決済法**: ステーブルコイン規制

## 4. 商習慣・契約慣行

### 営業プロセス
- **適合性原則**: 顧客の知識・経験・財産・取引目的に応じた商品提案
- **重要事項説明**: 法定書面の交付・説明義務
- **クーリングオフ**: 一定期間の契約解除権
- **個別対応**: 大口顧客・機関投資家向けは個別契約

### システム導入
- **長期検証**: 半年〜2年の POC・並行運用
- **規制対応**: 金融庁ヒアリング・FISC 監査
- **ベンダー選定**: 入札 + 多重審査
- **支払サイト**: 月末締め翌月末払い等が標準

### コンプライアンス
- **三線防御**: 第1線（営業）/ 第2線（コンプライアンス・リスク管理）/ 第3線（内部監査）
- **Fit and Proper**: 役員の適格性審査（金融庁）
- **行動規範**: 利益相反管理 / インサイダー取引防止

## 5. 典型的な失敗パターン

### 失敗 1: 規制違反による業務停止命令
- 不適切販売（顧客本位違反）/ AML 体制不備 / システム障害放置
- → legal-compliance-checker と金融庁ガイドラインの定期確認

### 失敗 2: システム障害の社会問題化
- 大規模障害でメディア・国会で取り上げられる事態
- → ヴォーゲルス Frugal Architect / 障害前提設計 / 多重化

### 失敗 3: AML/CFT 不備による FATF 評価低下
- 国際送金・暗号資産で疑わしい取引を放置
- → AI による継続モニタリング・SAR（疑わしい取引届出）の体制整備

### 失敗 4: FinTech との競争で本業侵食
- 決済・送金・与信で FinTech に顧客流出
- → 銀行 API 開放・BaaS 化・FinTech 協業

### 失敗 5: 「絶対儲かる」表現での金商法違反
- 投資商品の広告・説明で断定的判断を提供
- → 全マーケ素材を法務・コンプライアンス審査

## 7. 提案・分析時のチェックリスト（金融特化）

### 提案前（重要）
- [ ] **金融庁監督指針** の最新版確認
- [ ] **業態ライセンス**確認（銀行業 / 金商業 / 保険業 / 資金移動業等）
- [ ] **法務・コンプライアンス部門**との事前調整
- [ ] AML/CFT 体制の確認

### 提案中
- [ ] **適合性原則**遵守の設計（顧客属性に応じた商品）
- [ ] **断定的判断の提供禁止**（「絶対儲かる」「100%安全」等の禁止表現チェック）
- [ ] **重要事項説明**の組み込み
- [ ] **個人情報保護**＋ **金融分野特則**遵守
- [ ] **FISC 安全対策基準**準拠（システム提案時）

### 提案後
- [ ] 行政指導・規制変更の継続ウォッチ
- [ ] 顧客本位の業務運営方針との整合性
- [ ] 内部監査・第2線レビューの実施
- [ ] AML/CFT モニタリングの継続
