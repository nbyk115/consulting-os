## 9 セクション構造 - 主要論点

### §1 案件サマリ（30 秒版 SCQA + Governing Thought）

Situation: マーケ / アドテク業界の AI 化が加速し、代理店・事業会社の運用知識がコモディティ化する。Complication: 個別ツールは乱立するが「マーケ判断の OS」は未存在。Question: 誰がカテゴリファーストを取るか。Answer: ConsultingOS のマーケ部門 7 agent が、アドテク業界出身の水野氏ネットワークを配布チャネルとして即商用化できる唯一の状態にある。

Governing Thought 1 行: 「マーケ OS の間隙を、ジーニー OB ネットワーク経由で独占する 1000 万円」。

### §2 水野氏属性（FACT 限定、handoff §2 準拠）

ジーニー（東証プライム上場）役員 / 退任予定 / 退任後個人投資家予定、アドテク / マーケ SaaS 業界、ユーザーとの先輩関係 + 個人信頼ベース出資意向（以上全て FACT・ユーザー口頭確認 2026-05-08）。ConsultingOS マーケ部門 7 agent との直接ドメイン重複（INFERENCE）。金融業フレーム（投資助言業 / 銀行審査部 3 共通点 / 野村幸司氏）は一切不使用。

### §3 ConsultingOS 全体像（マーケ部門 7 agent 中核、27 agent 27 skill 体制）

6 部門 27 agent のうちマーケ部門（marketing-director / performance-marketer / seo-specialist / marketing-analyst / social-media-strategist / market-researcher / pr-communications）が水野氏ドメインと直接重複。「マーケ業界で使う AI コンサル OS」として位置づけ。技術基盤: Claude Code MCP / Opus 4.7 / 21 hook 物理化（FACT）。プロダクトバリューの外側分析として「マーケ業界の間隙選定の妥当性」を 1 スライド分確保。

### §4 GTM 戦略（N&Y Craft 第 1 リファレンス + 水野紹介マーケ案件）

2 段展開。Phase 0: N&Y Craft 関根さん案件（2026-05-09 提案・マーケ系 OEM）で ARR 実証 1 件目を確立。Phase 1: 水野氏の業界ネットワーク（ジーニー OB / アドテク / マーケ SaaS 周辺）経由で OEM 2-3 件を紹介導線化。CAC ゼロのネットワーク起点を構造化（属人営業ではなく紹介手数料覚書を契約形態として整備）。Phase 2: 国内マーケ業界カテゴリトップ集約。3 変数交点（コンサル経験 x OEM 配布チャネル x Anthropic Opus 4.7）が 2026 年初に重なる窓（INFERENCE）。

### §5 数値計画（ARR / NRR / CAC payback - マーケ SaaS 業界基準）

v3 の「軸 A 投資助言業 / 軸 B OSS CRM 二軸」PL は全廃。マーケ業界フレームで再設計。最低限の 4 KPI: OEM 月次 ARR（Phase 1: 2-3 件 x 20-60 万 = 40-180 万 / 月 INFERENCE）/ NRR 目標 110% 以上（INFERENCE）/ CAC payback（紹介ゼロ CAC を仮定した場合の回収月 INFERENCE）/ ランウェイ（1000 万円 x 最小固定費 12-15 万 / 月 = 67-83 ヶ月 INFERENCE）。ブレイクイーブン月を「XX ヶ月目」で数値明示。4 シナリオ（楽観 / 標準 / 悲観 / 致命）継承。

### §6 競合分析（Sierra / Agentforce / NotebookLM - マーケ視点更新）

v3 の 4 レイヤー構造は骨格を継承し、マーケ業界目線で内容を更新。L1: 大手コンサル（マーケ分野: dentsu / 博報堂 DY / サイバーエージェント等）/ L2: 個別マーケ AI ツール（Jasper / Copy.ai / HubSpot AI 等）/ L3: マルチエージェント（Sierra カスタマーサービス特化 / Salesforce Agentforce CRM 直結 / NotebookLM 情報整理特化）/ L4: 代理店内製 AI。「敵地で戦わない」軸: マーケ判断 OS として L3 の間隙（垂直統合 vs 汎用）を取りにいく構造。Secret Thesis 3 点（Thiel 適用）はマーケ業界文脈で差し替え。

### §7 リスク開示（マーケ業界フレームで再設計）

v3 の「投資助言業不認可リスク」は削除。新設 3 点: 1. Anthropic 依存（API 価格改定 / モデル deprecation - 対策: モデル ID 環境変数化 / Bedrock セカンダリ）/ 2. Sierra / Agentforce 侵食（大手テック vs 中堅コンサル OS - 対策: 垂直統合 + evolution-log 蓄積が参入障壁）/ 3. 個人依存（1 名兼務の属人化 - 対策: OS に 4 思想を書き込み、誰が起動しても同水準）。Pre-mortem 4 シナリオ（カーネマン式）を継承・マーケ案件版に更新。「銀行審査部共通点」「AGPL-3.0 リスク」「投資助言業 500 万円供託」は全廃。

### §8 マスク式 4 原則統合（評判資本主義の体現として Ask の前に配置）

思考原則 2「評判資本主義」を §8 の中核に据える: 現金は遅行指標、評判 / 知識 / 実証 ROI が真の資産（consulting-playbook.md L470 FACT）。水野氏の「個人を信じる 1000 万」はこの評判資本主義の体現であり、出資決定の構造的根拠として読める。思考原則 1 インセンティブ設計（漸進 vs 大胆比率宣言 / 失敗コスト上限）+ 実行原則 3 Unkind Truth（Kind Lie 排除・反証チェック物理化）+ 実行原則 4 アルゴリズム（5 ステップ順序適用）を事業運営規律として提示。4 思想統合（孫正義 / Huang / 佐藤裕介 / マスク）の運用順序も記載。

### §9 Ask（出資条件 + 次アクション）

v3 の「軸 A 規制取得フェンス資金 500 万 / 軸 A 申請費 500 万の 2 段設計」は全廃。新設計: 1000 万円の用途をマーケ業界 OEM 商用化に集中（Phase 0-1 の 18 ヶ月固定費カバー + N&Y Craft 受注 → 水野紹介 2-3 件受注の確実性を資金で買う構造）。J-KISS 推奨（バリュエーションキャップ要協議）は継承。出資契約形態の 3 択（普通株 / 種類株 / J-KISS）も継承。次アクション 3 分岐（合意 / 条件付き / 不成立）は継承。MEDDPICC 資格化チェック（proposal-writer エージェント定義の必須ゲート）を巻頭付近に内包。

## 反証チェック結果

- Step 1 自己反証: §6 競合の Sierra / Agentforce 選定はマーケ業界視点から適切か。Sierra はカスタマーサポート特化（FACT）でマーケ OS との競合は間接的、より直接的な競合（Jasper / HubSpot AI 等）を L2 に追加すべき。§5 数値は全 INFERENCE、水野氏が「エビデンスなき数値」と判定するリスクあり: N&Y Craft 受注前段階であり「Phase 0 実証前の仮説」と明記して説明する構造が必要。
- Step 2 構造反証: §8 マスク式をリスク開示の後（§9 直前）に配置することで「難しいことを言った後の思想武装」に見えるリスクがある。対案として §8 を §1 サマリ直後に移動し、投資判断の哲学フレームとして先出しする構造も検討する価値あり（2 案のうちユーザー選択推奨）。MEDDPICC ゲートは §9 Ask 内に内包予定だが、Champion（水野氏）が退任前 / 退任後どちらの立場で出資判断するか（Economic Buyer と Champion の一致 / 不一致）が未確認（INFERENCE）。
- Step 3 実用反証: oem-packaging-mizuno.md の「金融 OEM 6 ワークフロー」記述は水野氏が投資家として活用するシナリオ設計のままになっている箇所（§5-B 株式取引プロンプト 7 種）がある。v4 では「水野氏が自分の投資業務に使う」ではなく「水野氏が保有するネットワーク先の代理店 / 事業会社に OEM 提供するための ConsultingOS マーケ部門」として再フレーミングが必要（handoff §4 旧フレーム→新フレーム準拠）。
- Step 4 リスク即潰し: 「v4 に向けた構造判断」は本ターン完了。物理的な v4 ファイル作成は次アクション。本ターン内で即潰せるリスクとして「oem-packaging-mizuno.md §5-B の金融投資家フレーム残存」を確認した: 同ファイルは次セッションで v4 ファイル群と同時に更新が必要（構造的に本セッション単独での書き直しは依頼範囲外のため、例外規定該当・ただし今確認済みとして記録）。
