> **このファイルは ConsultingOS 自体の ICP 兼テンプレート**。マーケ / セールス / コンテンツ制作・カスタマーサクセス系エージェントは必ずこれを読み込んでから作業する。新規クライアント案件では、このテンプレートを `<project>/ICP.md` にコピーしてクライアント固有の ICP に書き換える。

## 2. ConsultingOS 自体の ICP（参考デフォルト）

### Primary ICP: マルチエージェント設計を要する事業会社・コンサル・代理店

**属性**
- 規模: 従業員 50-500 名 / 売上 5-100 億円
- 業界: 広告代理店 / コンサルティングファーム / SaaS 事業会社 / D2C ブランド
- 役割: 経営層・戦略責任者・マーケティング責任者・プロダクト責任者
- 地域: 日本国内中心（グローバル展開意欲あり）

**痛み（Pain）**
- 提案書 / 戦略文書のクオリティをスケールさせられない（属人化）
- AI を使いたいが、単発プロンプトでは品質が安定しない
- 複数部門（戦略 / 開発 / クリエイティブ / マーケ）の連携が分断されている
- 経営判断と実装の間に翻訳レイヤーが必要
- 機密性の高いクライアント情報を Cloud LLM に丸投げできない

**ジョブ（Job to be Done）**
- 提案書・戦略分析・LP・セールス資料を **属人化せず量産**したい
- AI を「**規律ある専門家チーム**」として使いたい
- クライアント案件ごとに**ブランドと文脈を維持**したい
- 経営判断（PL・市場構造）から実装（コード・デザイン）まで**一気通貫**で動かしたい

**評価軸（Buying Criteria）**
- 設計の網羅性（経営〜実装の統合度）
- セキュリティ多層防御（Layer 0/1/2）
- 反証モードによる品質ガード
- 実証された参照実装の有無
- カスタマイズ容易性

---

## 3. 非ターゲット（明示的に外す）

> 佐藤裕介流: 「**敵地で戦わない**」。以下は ConsultingOS の差別化が効かない領域。

- **個人開発者の単発ツール用途**: Cursor / Cline / GitHub Copilot のような IDE 拡張で足りる
- **エンタープライズの完全カスタム要件**: 独自セキュリティ要件 / オンプレ専用環境
- **完全初心者**: Claude Code 自体の基礎が必要（教科書系記事の対象）
- **AI 倫理・コンプライアンスが極端に厳しい業界**: 防衛・医療など、独自法規制対応が必要な領域

## 4. ペルソナ別利用文脈

### Persona A: 戦略コンサルタント（PM / シニア）
- **使用シーン**: 提案書執筆 / 競合分析 / クライアント PL 試算
- **起点エージェント**: `strategy-lead` / `proposal-writer` / `competitive-analyst`
- **重視するもの**: 反証モード、引用ソースの厳密さ、`first-principles-breakdown`
- **避けたいもの**: ハルシネーション、空疎な提案

> **自社事業による検証中**: 「よるのことば」（占いサービス）が ICP のスマホ占い主要利用層仮説と完全整合、運用知見を Persona B（マーケ）/ Persona C（クリエイティブ）にもフィードバック予定（README 自社事業セクション参照）

### Persona B: マーケティング責任者（事業会社 CMO / 代理店プランナー）
- **使用シーン**: チャネル戦略 / SEO / 広告運用 / GEO 設計
- **起点エージェント**: `marketing-director` / `seo-specialist` / `content-strategist`（GEO / AIO、旧 agentic-content 機能統合済）
- **重視するもの**: 実データ接続（toprank, /last30days）、ROI 数値化
- **避けたいもの**: 表面的なナレッジ提案、データ無根拠の推奨

### Persona C: クリエイティブディレクター（広告代理店 / D2C ブランド）
- **使用シーン**: LP 制作 / セールス資料 / SNS コンテンツ / 動画
- **起点エージェント**: `creative-director` / `sales-deck-designer` / `frontend-dev`
- **重視するもの**: ブランド一貫性、`DESIGN.md`、Hyperframes での動画量産
- **避けたいもの**: パクリ系ツール（ai-website-cloner 等は禁止）

### Persona D: テック / プロダクト責任者
- **使用シーン**: アーキテクチャ設計 / コードレビュー / AI 機能統合
- **起点エージェント**: `tech-lead` / `fullstack-dev` / `ai-engineer`
- **重視するもの**: セキュリティ Layer 0/1/2、反証モード、`engineering-playbook`
- **避けたいもの**: 単発のコード生成（規律のない出力）

## 5. 価値提案（Why ConsultingOS）

| 競合カテゴリ | 競合の強み | ConsultingOS の差別化 |
|---|---|---|
| 単発プロンプト集 | 即使える | 反証モード + 自己進化 + 多層防御で**長期運用品質**が桁違い |
| 個別 AI ツール（Cursor / v0 等） | 領域特化で深い | **6部門統合**による経営判断 → 実装の一貫性 |
| 大手コンサルファーム | 人的リソース | **再現可能な構造化された手法**、属人化排除 |
| 自前 LLM 構築 | 機密対応 | **すぐ動く** + Claude Code 公式エコシステム + Layer 0/1/2 |

**コア価値**: 「**経営判断統合支援を AI 駆動で再現可能にする規律 OS**」（唯一性は PMF 検証後に主張、現時点は INFERENCE）

## 9. 商業展開ナレッジ（実需顕在化次第で商品化検討）

> 出典: Damian Player「ChatGPT 5.5 found my first clients on Google Maps in 10 minutes」/ マネジメント記憶構造化フレーム / 2026-05-02 学習。
> 全部実装ではなく、案件問い合わせが来たら個別評価。

### 9.1 日本ひとり社長向け展開可能ターゲット業種（Secondary ICP 候補）

Vertical AI 適合度の評価軸（出典: 自民党 AI・web3 小委員会 第 7 回会合 柴田直樹氏提言、https://note.com/akihisa_shiozaki/n/n3bbe75fa841c）: 業界知見蓄積度（OJT 密度・暗黙知の濃さ）× Vertical AI 学習データ生成可能性（業務ログ・判断記録のデジタル化可能性）で高 / 中 / 低を判定（INFERENCE: 基準適用による個人評価、実態調査で要検証）。

2040 年人材需給ポジションの補足観点（出典: 経産省「2040 年の就業構造推計（改訂版）」2026 年 3 月、FACT）: AI・ロボット利活用人材の需要 782 万人 / 供給 443 万人 / 不足 339 万人、事務職余剰 440 万人。下表「人材需給ポジション」列はこの推計に基づく業種別の需給構造を示す（INFERENCE: 推計の業種別内訳は個別確認が必要。出典 URL: https://ledge.ai/articles/2040_ai_robot_human_resources_shortage_meti / https://www.nikkei.com/article/DGXZQOUA1948E0Z10C25A5000000/）。補完文献として MUFG 三菱 UFJ 信託銀行 試算研究所「人手不足の実状と高圧経済政策について」（前編・後編）を参照（INFERENCE: AI・ロボティクス浸透前提の職種別・学歴別余剰/不足分析および高圧経済政策フレームを提供。詳細は marketing-research-playbook.md §9.4.1 MUFG 論点。出典 URL 要確認: tr.mufg.jp/shisan-ken/pdf 以下）。

| 業種 | 特徴 | 受注形態（候補） | Vertical AI 適合度 | 2040 年人材需給ポジション（INFERENCE） |
|---|---|---|---|---|
| リフォーム / 外壁塗装 | 単価高（工事 1 件 100-500 万円）/ ホームページ古いか無し | サイト制作 8-25 万円 + 月次保守 1.5-3 万円 | 中（職人技・現場判断の暗黙知あり。ただし業務ログのデジタル化が遅れており学習データ化が難） | 人手不足継続（建設・土木系は技能人材不足が深刻。AI による施工管理補助ニーズが顕在化する見込み） |
| エアコンクリーニング / 不用品回収 / 害虫駆除 | 季節需要 / Google Maps 評価が直接受注に直結 | レビュー返信代行 月 3-8 万円 | 低（作業手順が比較的標準化。業界固有の暗黙知蓄積が薄く Vertical AI 優位性が出にくい） | 中立（単純作業はロボット化が進む一方、現場対応の人的サービスは一定量残存。事務職余剰 440 万人の再配置先候補） |
| 歯科 / 整体 / 美容院 | レビュー対策 = 売上直結（食べログ / Google）/ 返信時間なし | 月次レビュー対策 月 3-8 万円 | 中（歯科は診断・保険請求の暗黙知密度が高く Vertical AI 余地あり。整体・美容は施術記録のデジタル化次第） | 医療・介護系は人手不足継続（高齢化による需要増と医療従事者不足の交差点。ConsultingOS 提供価値：バックオフィス AI 化で診療・施術に集中できる環境を提供） |
| 士業事務所（税理士 / 行政書士 / 社労士）| ホームページ更新時間なし / SEO 知見ゼロ | サイト + コンテンツ月次更新 月 5-10 万円 | 高（法律・税務の専門知識 + OJT 蓄積が濃密。業務記録のテキスト化がしやすく Vertical AI のファインチューニングデータが生成されやすい） | 人手不足セクター（専門知識を要する高度業務は AI 代替困難。同時に AI 活用で業務処理量を拡大できる「高適合 + 人材不足 = ConsultingOS 提供価値最大」の最優先ターゲット） |
| 葬儀社 | 競合差別化困難 / Web 経由問い合わせ少 | 検索対策 + サイト改修 月 5-10 万円 | 低（業界知見は地域密着型で汎用化しにくい。データ量・標準化の難度から Vertical AI 展開規模が限定的） | 中立（少子化による市場縮小と高齢化による需要増がクロスするニッチ。事務職余剰人材の吸収余地は小さく、AI 投資インセンティブも低い） |

判定軸: ★4.0 以上 + レビュー 20 件以上 + ホームページ無し or 古い の 3 条件で Google Maps から候補抽出。

### 9.3 営業手法の規律（既存内包の強化）

- 「営業」ではなく「成果物プレビュー」: AI で 1 時間で完成形を作って見せる、ご提案させてください方式は禁止
- アウトリーチは多チャネル（メール一辺倒禁止）: SNS DM / X DM / LINE 公式 / 郵送（赤入れ PDF）の組み合わせ
- 1 日 500 通自動メールは禁止（特定電子メール法違反）
- スクレイピングは ToS 遵守ツールのみ（Apify / Octoparse 公式 SDK 範囲内、Outscraper / PhantomBuster は規約確認後）

### 9.4 取り込み禁止事項

- 1 日 500 通の自動コールドメール（特定電子メール法違反）
- Google Maps / 食べログ等の ToS 違反スクレイピング（不正アクセス禁止法 3 条グレー）
- bot 偽装ツール（Scrapling 系、別途 evolution-log で禁止判定済）
