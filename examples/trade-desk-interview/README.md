# 実例：Trade Desk インタビュー 翻訳 ＋ 知見シェア要約

Claude Code のマルチエージェント／スキル活用によるコンサルティング業務支援の実例。
英語インタビュー記事を日本語に翻訳し、さらに社内ナレッジシェア用の要約ドキュメントまで一貫して生成した記録。

---

## 案件概要

| 項目 | 内容 |
|---|---|
| 案件種別 | 業界記事翻訳＋ナレッジシェア要約 |

## 成果物

- [`translation-ja.md`](./translation-ja.md) : 全文日本語訳（Q&A 6問、忠実訳）
- [`summary-ja.md`](./summary-ja.md) : 知見シェア用要約（TAKEAWAY 5点＋示唆表＋ディスカッション論点）

## 使用したエージェント／ツール構成

### 翻訳フェーズ：オーケストレーター＋並列翻訳サブエージェント

```
┌─────────────────────────────────┐
│  オーケストレーター（メインAgent）      │
│  - タスク分割                      │
│  - サブエージェント起動              │
│  - 成果物統合                      │
└──────────┬──────────────────────┘
           │
    ┌──────┴──────┐
    ▼             ▼
┌────────┐   ┌────────┐
│ 翻訳者A │   │ 翻訳者B │   ← general-purpose サブエージェント
│ 前半3問 │   │ 後半3問 │      並列実行
└────────┘   └────────┘
```

- Agent tool（`general-purpose`）を2つ並列起動
- 翻訳タスクを前半／後半に分割し、各サブエージェントに業界用語ガイドライン付きで指示
- メインエージェントが2つの成果物を統合し、ブランドセーフティ／インベントリ等の用語統一を確認

### 要約フェーズ：メインエージェント直接実行

- 翻訳フェーズで全文コンテキストを保持済みのため、分割不要と判断
- 構造化テンプレート（背景／TAKEAWAY／示唆表／ディスカッション論点）に沿って要約生成

### スキル活用余地（次回以降の改善ポイント）

- `simplify` スキル：翻訳文の冗長表現整理に活用可能
- レビュー専用サブエージェント：翻訳精度の第三者チェック層として追加可能
- `Plan` エージェント：より大規模な翻訳プロジェクトでは最初にタスク分割設計を任せられる

## ワークフロー記録

1. 起動：ユーザー指示「［翻訳］オーケストレーター起動」
2. 翻訳：2サブエージェント並列実行 → 統合
3. 保存：`translation-ja.md` にコミット
4. 要約依頼：ユーザー指示「会社知見シェアようにようやくして」
5. 要約生成：構造化テンプレートで TAKEAWAY 抽出
6. 実績化：`examples/trade-desk-interview/` 配下に整理

## 再現手順（同種案件のテンプレート）

1. ソーステキストを受領
2. ボリュームを確認し、分割翻訳の要否を判断（目安：2,000 words 以上で分割推奨）
3. メインエージェントが翻訳サブエージェントへガイドライン付きで指示
   - 対象読者／トーン／用語ポリシー／出力フォーマットを明示
4. サブエージェント並列実行
5. メインエージェントが統合・用語統一・ファイル書き出し
6. ナレッジシェア要約が必要な場合：背景／TAKEAWAY／示唆／論点の4層構造で生成

## 学び

- 「オーケストレーター起動」のような明示的な指示は字義通り実行すべし
  - 短いタスクでもサブエージェント構成を組む価値がある（並列化＋品質保証層）
- 翻訳品質は"業界用語ガイドライン"の事前提示で大きく変わる
  - DSP／インベントリ／ブランドセーフティなどの定訳を指示に含めることで揺れを抑制
- 要約はメインエージェントが直接やる方が速い場合も多い
  - 翻訳コンテキストを既に保持しているため、サブエージェント起動は冗長になりうる
- 実績化の意義：エージェント構成・プロンプト設計・ワークフローを残すことで、次回以降のテンプレート化が可能
