> hotice deck で実証済み。日本の代理店・エンタープライズ向け B2B 提案資料の標準構成。**このテンプレートをコピーして、各ページの中身をクライアント案件に合わせて書き換える**。

## 各ページの設計原則（必読）

### 1ページ 1メッセージ
複数のメッセージを 1 ページに詰め込まない。読者が 5 秒で要点を掴める粒度に。

### 役割が通っているか
**静 → 知 → 強 → 信 → 実務 → 決** の流れが通っているか。役割の重複・抜けがあれば再構成。

### 自社紹介より先に売り込みを始めない
P2 が会社紹介、P5 以降がポジショニング。**P2 でいきなりサービス紹介はしない**。

### 敬語軸の統一
- 「代理店様 / 御社 / 〜いただけます」
- ❌「〜できます / 〜組めます / 代理店が」（上から目線）

### 業界略語は展開
- ❌ MCN → ✅ クリエイター事務所
- ❌ MKT → ✅ マーケティング
- ❌ SNS → ✅ ソーシャル（または個別プラットフォーム名）

## ページ別の典型パターン

### P1 カバー: 二項動詞型タグライン
```
"買いたい" を、売上に変える
```
- 「X を Y に変える」の構造
- 抽象語（気づき・体験・共感）禁止

### P5 真の課題: 「指標を疑う」翻訳
```
表面: CPA が高くて改善したい
真因: そもそも CPA 改善で売上が伸びる構造ではない
```

### P8 競合比較: 領域限定
```
○: インフルエンサー領域での独自性
×: 総合広告領域での横並び比較（自社が不利）
```

### P17 PL 試算: 前提条件と数字
```
前提: 月額○○円 × ○○ヶ月 = 投資総額
効果: ROAS ○○倍 → 粗利 ○○○万円増
ブレイクイーブン: 第○ヶ月
```

## 省略可能ページ

クライアントの状況によって省略可:
- P3 業界構造（既に共有されている場合）
- P11 数字実績（守秘契約上厳しい場合 → P10 ケースに統合）
- P12 クライアントの声（実績ページに統合可）
