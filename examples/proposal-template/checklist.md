# 提案書 提出前チェックリスト（佐藤裕介モード反証）

> 提出前に必ず通す品質ゲート。1項目でも `❌` があれば提出禁止 → 修正してから再チェック。

---

## 1. コピー品質

- [ ] タグライン（P1）が「X を Y に変える」の二項動詞型になっている
- [ ] 抽象語（気づき・体験・共感・寄り添う）を**定義なしで使っていない**
- [ ] 業界略語（MCN/MKT/SNS 等）を**全てカタカナ展開**している
- [ ] 上から目線の動詞（〜できます／〜組めます／代理店が）を全て「〜いただけます／代理店様」に置換済み
- [ ] 無根拠の最上級（「最も」「業界最大級」「唯一」）を全て**実証可能表現に置換**
- [ ] 各ページの最重要1文を「」で強調している (Hard Rule 16 ① 準拠、太字記法 禁止)
- [ ] 1段落 3文以内
- [ ] フィラー（「重要なのは」「今日の世界では」）禁止
- [ ] ヘッジ（「かもしれない」「おそらく」「〜のようだ」）禁止

---

## 2. ハルシネーション・出典

- [ ] **全ての数値**にソース URL + 発行日が紐づいている
- [ ] 派生数値（「80%が…」など）が一次ソースから**導出可能**
- [ ] 競合情報が**公式 / IR / 主要紙ソース**である
- [ ] **代理店ヒアリング等の非公開データ**を資料に**載せていない**
- [ ] 上場企業のケースは**匿名化**されている（フェアディスクロージャー配慮）
- [ ] クライアント名は**一次公開ソース**で確認済み（許諾なき実名禁止）

---

## 3. 競合比較（P8）

- [ ] 「非公表 / 要相談」が**半数以上の行を削除**済み
- [ ] **自社が負ける軸**は載せていない（または軸名を変えて文脈化）
- [ ] **自社の弱みを自分から surface していない**（留意点行なし）
- [ ] 商品名（MTA / GOAT / AnyTag 等）ではなく**解釈した強み**を主文に書いている
- [ ] 競合カテゴリによって**領域を限定**している（例：「インフルエンサー領域での」強み）

---

## 4. PL試算（P17）

- [ ] **粗利インパクト**を数字で示している（「大幅に」ではなく「30%改善」「○○○万円増」）
- [ ] **ブレイクイーブン**月を明示
- [ ] **前提条件**（投資額・期間・ROAS 等）が箇条書きで明示
- [ ] 楽観シナリオ・現実シナリオ・保守シナリオの**3パターン**が並列されている（推奨）

---

## 5. レイアウト・実装

- [ ] **横揃え**: 複数カードの proof box / KPI / CTA が横一線にある
- [ ] **縦揃え**: 下段（.srcLine / .caseNote / .footnote）が下端に pin
- [ ] **改行**: 日本語が文節境界で改行されている（`word-break: auto-phrase` 適用）
- [ ] **文字切れ**: `transform: scale()` や `bottom: -px` で突き抜けた要素なし
- [ ] **重複情報**: 見出しに含まれる数字を直下の大数字で繰り返していない
- [ ] **日本語字形**: `lang="ja"` 設定済み、Yu Gothic 系フォント、中国字形フォールバックなし

---

## 6. 情報設計

- [ ] **1ページ 1メッセージ**になっている
- [ ] 自社紹介（P2）より**先に売り込みを始めていない**（P5以降がポジショニング）
- [ ] 各ページの役割（**静→知→強→信→実務→決**）が通っている

---

## 7. 佐藤裕介オーナー反証（最終チェック）

- [ ] **PL に落ちる**提案になっている（粗利インパクト数字あり）
- [ ] **市場構造から**入っている（個社の SWOT より先に業界の構造）
- [ ] **アセット優先**（消耗施策ではなく積み上がる価値訴求）
- [ ] **敵地で戦わない**（自社が負ける土俵に乗っていない）
- [ ] **新市場挑戦**: 「参入する力があるのに挑戦しない」最大リスクが提案に含まれているか（該当案件のみ）

---

## 8. 小野寺信行オーナー反証（マーケ系のみ）

- [ ] **指標を疑う**: クライアント主張 KPI を本当の課題に翻訳済み
- [ ] **フロー × ストック統合**: 単発施策と資産蓄積施策をセット提案
- [ ] **文脈設計**: ターゲットを「いまどんな状況にある人か」で定義
- [ ] **目的別 KPI**: ブランディング・育成・獲得・リテンションで異なる KPI 設計
- [ ] **1stParty データ中心**: 外部データ依存はリスクとして明示

---

## 9. 反証モード（必須）

- [ ] **Step 1（自己反証）**: 自分の結論に対する反論・反例を最低3つ挙げた
- [ ] **Step 2（構造反証）**: ロジック飛躍・数値妥当性・抜け漏れを検証した
- [ ] **Step 3（実用反証）**: 「クライアントがこれを見たらどう反応するか」をシミュレートした
- [ ] **残存リスク**: トリプルチェック後も残るリスクを明示している

---

## 10. レンダ・配布

- [ ] HTML プレビューで全ページを目視確認
- [ ] PNG / PDF レンダで崩れがないか確認（`node render.mjs`）
- [ ] PPTX 必要時は python-pptx で生成、フォント中国字形フォールバックなしを確認
- [ ] CDN 配布時は `rawcdn.githack.com/<repo>/<commit-sha>/...` 形式（キャッシュ回避）
