# 価格根拠＋値引き交渉時の防御線（自分用）

各価格の根拠を可視化し、関根さんから値引き交渉が来た時の押し返し材料を整理する。
**佐藤裕介フレーム原則**：価格の根拠を自分が説明できないと、必ず値切られる。

---

## 1. 価格設計の4軸（再掲）

副業OEMの値付けは、感覚で出すと必ず外す。以下4軸の交差点で決める：

| 軸 | 内容 |
|---|---|
| 軸1：**顧客の支払い能力** | N.Y.CRAFT がいくら払えるか（月商の5〜10%が天井） |
| 軸2：**顧客の価値** | OS が N.Y.CRAFT にもたらす経済的価値 |
| 軸3：**自分のコスト** | 構築・運用工数 × 副業時給 |
| 軸4：**代替手段の価格** | NotebookLM・他ツール・人材採用との比較 |

---

## 2. プロジェクトフィー：70万円

### コストベース計算

| ステップ | 工数 |
|---|---:|
| 関根さんインタビュー（5h × 4回） | 20h |
| 既存資料の取込み・整理 | 10h |
| コアスキル定義（3領域） | 15h |
| 基本テンプレ構築（5本） | 20h |
| 試運転・チューニング | 10h |
| キックオフ研修準備＋実施 | 5h |
| **合計** | **80h** |

副業時給 1.5万円換算：**80h × 1.5万 = 120万円**（標準価格）
副業時給 2万円換算：**80h × 2万 = 160万円**（プレミアム価格）

### なぜ標準120万を70万に圧縮したのか

**戦略的ディスカウント = 30〜50万円分を「リファレンス取得への投資」として計上**

| 自分が支払う対価 | 関根さんから受け取る対価 |
|---|---|
| プロジェクトフィー 30〜50万分の圧縮 | N.Y.CRAFT 名出しでの事例公開権 |
| | 構築プロセス・成果のケーススタディ化権 |
| | 他社OEM営業時のリファレンスコール協力（年2回まで） |
| | 関根さんver で得た知見の他案件への再利用権 |

→ ここを **「ただの値引き」ではなく「等価交換」** として説明できることが大事。

### 70万を切ろうとされた場合の防御線

| 関根さんの言い方 | 押し返しスクリプト |
|---|---|
| 「もう少し安くならない？」 | 「すでに通常120万のところを、リファレンス案件として70万まで圧縮しています。これ以上は工数的に難しいです」 |
| 「最初は安く始めて、効果が出たら値上げで」 | 「Phase 1 / Phase 2 の構造で、まさにそれをやっています。Phase 1 が70万、Phase 2 で必要に応じて追加50〜80万」 |
| 「分割払いで」 | 「着手前一括が原則です。構築工数の大半を最初の3ヶ月で投下するため」 |

**最低ライン：70万**。これを下回るなら受けない（時給が1万円以下になり副業破綻）。

## 3. ランニング：月12万円

### コストベース計算

| 月次タスク | 工数 |
|---|---:|
| 月次レビュー会（90分 + 準備30分） | 2h |
| 品質チェック（テンプレ・出力物） | 2h |
| アップデート反映 | 2h |
| Slack/メール質問対応 | 1.5h |
| **合計** | **7.5h** |

副業時給 1.6万円換算：**7.5h × 1.6万 = 12万円**

### N.Y.CRAFT の支払い能力との整合

- 推定月商：80〜400万（クライアント数 × 単価から逆算）
- 月12万 = 月商の **3〜15%**
- → 月商200万以上なら 6%以下で済む。**重さを感じない比率**

### 既存業務委託 月5万との心理的距離

- 月5万 → 月12万 = **2.4倍**
- 心理的に呑める範囲（一般に3倍以内が「同じ関係性の延長」と感じられる距離）
- これを超えると「全く別物の高額案件」という認知になり、慎重姿勢になる

### 値引き交渉が来た場合の防御線

| 関根さんの言い方 | 押し返しスクリプト |
|---|---|
| 「月10万にならない？」 | 「月10万にする場合、工数上限を月6時間に絞らせていただきます。レビュー頻度・対応範囲が下がりますが大丈夫ですか？」 |
| 「成果が出るまで月5万でお試し」 | 「お試し期間として最初の2ヶ月のみ月8万 + 工数上限5h でいかがでしょう。3ヶ月目から通常料金に戻します」 |
| 「成果報酬だけで運用したい」 | 「成果報酬だけだとこちらのリスクが大きすぎます。最低限の運用工数を担保するベースが必要です」 |

**最低ライン：月10万 + 工数上限6h**。これ以下は受けない。

## 4. 成果報酬：通常5% / 再販15%

### 通常受注 5% の根拠

**業界相場との整合**：

| ケース | 相場 | 採用 |
|---|---:|:---:|
| 紹介料 | 5〜10% | — |
| ライセンス料（売上ベース） | 3〜10% | ◯ |
| 代理店マージン | 10〜30% | — |
| プロダクトのレベニューシェア | 5〜20% | — |

**OS の貢献度評価**：
- N.Y.CRAFT のビジネスは **関根さんの営業力・関係性で売れている**
- OS は「業務効率化＋アウトプット品質向上」の補助ツール
- OS なしでも売れるが、OS があるとスケールできる
- → **OS は補助役、主役じゃない** → 5%が妥当

10% を要求しないのはなぜか：
- 副業フレーム＋関根さんとの長期関係を考えると 10% は攻めすぎ
- 関根さんが呑まない可能性が高く、契約全体が流れるリスク
- 5% でも年間数十万円は乗る → 十分な収益貢献

### 再販 15% の根拠

OS自体が商品となる場合：
- N.Y.CRAFT が他社に OS を販売／OEM 提供する
- → **OS は商品の主役そのもの**
- → 高めの料率が妥当

業界相場：プロダクトのレベニューシェアは 5〜20%、商品としての貢献度が高ければ高め。
**15% は中央値**。20% は強気すぎ、10% は控えめすぎ。

### 値引き交渉が来た場合の防御線

| 関根さんの言い方 | 押し返しスクリプト |
|---|---|
| 「成果報酬は無しで」 | 「ベース料金を上げる必要があります。プロジェクト100万・ランニング月15万にしてはいかがでしょう」 |
| 「5%は高い、3%で」 | 「3%にする場合、対象期間を24ヶ月→12ヶ月に短縮してはいかがでしょう」（実質的にこちらの取り分減らない） |
| 「再販15%は取りすぎ」 | 「再販時はこちらが OS の品質責任を負います。15%は業界平均で、これ以下にすると品質保証の根拠がなくなります」 |

**最低ライン：通常3% / 再販10%**。これ以下なら成果報酬抜きでベース料金を上げる。

## 5. 全体経済性のクロスチェック

### 自分側

| 項目 | 金額 | 工数 |
|---|---:|---:|
| プロジェクトフィー | 70万 | 80h |
| ランニング | 月12万 × 12 = 144万 | 90h |
| 成果報酬（仮：N.Y.CRAFT の OS活用売上 1,000万 × 5%） | 50万 | 5h |
| **計（業務委託除く）** | **264万 / 年** | **175h / 年** |
| 業務委託（既存） | 月5万 × 12 = 60万 | （別カウント） |
| **総計** | **324万 / 年** | |

副業時給換算：264万 / 175h = **時給 1.5万円**
業務委託込み：324万 / 175h = **時給 1.85万円**（業務委託工数を含めずに計算した場合）

→ **副業として成立ライン**。ただしギリギリなので、工数超過させないことが極めて重要。

### N.Y.CRAFT 側

| 項目 | 金額 |
|---|---:|
| プロジェクトフィー | 70万 |
| ランニング 月12万 × 12 | 144万 |
| 業務委託（既存）月5万 × 12 | 60万 |
| 成果報酬（仮）| 50万 |
| **合計** | **324万 / 年** |

仮に N.Y.CRAFT 月商200万（年商2,400万）なら、年商の **13.5%**。
やや重いが、**OS が事業の中核に入るなら正当化可能**な比率。

仮に月商300万（年商3,600万）なら 9%。**より楽な比率**。

→ N.Y.CRAFT の実際の事業規模次第で、関根さんの呑みやすさが変わる。
打合せで月商感覚を聞き出せると価格交渉の精度が上がる。

## 6. 値引き交渉が来た時の総合判断ロジック

### 必ず守るライン（譲ったら副業破綻）

| 項目 | 死守ライン |
|---|---|
| プロジェクトフィー | 70万 |
| ランニング | 月10万 + 工数上限6h |
| 成果報酬（通常） | 3% |
| 成果報酬（再販） | 10% |
| 知財所有権 | 全てこちら帰属（譲歩ゼロ） |

### 譲歩可能なオプション（こちらの利益を守りつつ呑む形）

1. **支払いタイミング調整**：分割払い（ただし金利分上乗せ）
2. **Phase 1 のスコープ縮小**：基本テンプレ5本→3本、構築期間短縮
3. **工数上限の引き下げ**：月8h → 月6h（その分料金も下げる）
4. **成果報酬対象期間の短縮**：24ヶ月 → 18ヶ月（料率は据え置き）
5. **お試し期間の設定**：最初の1〜2ヶ月のみ料金圧縮、その後通常料金

### 撤退ラインに達した場合

死守ラインを下回る要求が出て、関根さんが折れない場合：

> 「今回はお見送りさせてください。N.Y.CRAFT の規模感とご期待される水準のバランスで、
> 私どもがご提供できる範囲とギャップがあります。
> 業務委託は現状通り続けさせていただきますので、
> またタイミングが合えばお声がけください」

→ **副業の最大の損失は「断れずに受けた合わない案件」**。
→ 業務委託の関係は維持しつつ、OEM契約だけお断りする選択肢を残しておく。

## 7. 価格の前提が崩れる場合の再評価トリガー

以下のいずれかが起きたら、価格設計を見直す：

- N.Y.CRAFT の月商が想定より大幅に上下（半分以下 or 倍以上）
- 関根さんの構築への協力度が低い（インタビュー時間が確保できない等）
- Phase 1 構築工数が80hを大幅に超える見込みになった（120h以上）
- ランニング工数が連続3ヶ月で月10h超え
- 成果報酬の対象範囲で揉め事が連続発生

→ いずれかが起きたら、契約更新時 or 緊急協議で再交渉する。
