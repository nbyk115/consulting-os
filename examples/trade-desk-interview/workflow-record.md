## エージェント／スキル構成

```
┌─────────────────────────────────┐
│  オーケストレーター（メイン）       │
│  - タスク分割                    │
│  - サブエージェント起動            │
│  - 成果物統合・用語統一           │
└──────────┬──────────────────────┘
           │
    ┌──────┴──────┐
    ▼             ▼
┌────────┐   ┌────────┐
│ 翻訳者A │   │ 翻訳者B │   ← general-purpose 並列
│ 前半3問 │   │ 後半3問 │
└────────┘   └────────┘
```

- **使用スキル（当時／本基盤確立前）**: なし（基盤未整備のため、口頭指示で代用）
- **本基盤適用時の対応スキル**:
  - `client-context-load` 相当：依頼者プロファイル＋業界用語ガイドラインの事前提示
  - `competitor-scan` 系の構造：並列サブエージェント運用パターン
- **使用サブエージェント**: `general-purpose` × 2（並列翻訳）
- **並列／逐次の判断理由**:
  - 翻訳：6問 × 平均ボリュームで分割効果あり → 並列
  - 要約：翻訳コンテキスト保持済 → 逐次（メイン直接実行が最速）

## ワークフロー記録

1. **起動**：ユーザー指示「［翻訳］オーケストレーター起動」
2. **翻訳**：`general-purpose` 2並列実行（前半3問／後半3問）
   - 業界用語ガイドライン（DSP、インベントリ、ブランドセーフティ等の定訳）を指示に含める
3. **統合**：メインエージェントが2成果物を結合・用語統一
4. **保存**：[`translation-ja.md`](./translation-ja.md) コミット
5. **要約依頼**：ユーザー指示「会社知見シェアようにようやくして」
6. **要約生成**：背景／TAKEAWAY／示唆表／ディスカッション論点の4層構造
7. **実績化**：`examples/trade-desk-interview/` 配下に整理・コミット

## 学び（次回テンプレ化のための気づき）

- **「オーケストレーター起動」のような明示的な指示は字義通り実行すべし**
  - 短いタスクでもサブエージェント構成を組む価値（並列化＋品質保証層）
- **翻訳品質は"業界用語ガイドライン"の事前提示で大きく変わる**
  - 定訳を指示に含めることで揺れを抑制
  - → 本基盤の `client-context-load` で「ドメイン固有語彙」セクションとして恒常化
- **要約はメインエージェントが直接やる方が速い場合も多い**
  - 翻訳コンテキストを既に保持しているため、サブエージェント起動は冗長
  - → 「コンテキスト保持済の場合は直接実行」をワークフロー判断ルールとして明文化
- **実績化の意義**：エージェント構成・プロンプト設計・ワークフローを残すことで、次回以降のテンプレート化が可能
- **本基盤確立後の改善余地**：
  - 翻訳系スキル `translation-orchestrator` を将来追加候補（実需が複数件出てから）
  - レビュー専用サブエージェント（翻訳精度の第三者チェック層）

## 再現手順（同種案件のテンプレート）

1. `templates/intake/client-context.md` を埋める（社内案件でも省略しない）
2. ソーステキスト受領、ボリューム確認（目安：2,000 words 以上で分割推奨）
3. メインエージェントが翻訳サブエージェントへガイドライン付きで指示
   - 対象読者／トーン／用語ポリシー／出力フォーマットを明示
4. `general-purpose` 並列実行
5. メインエージェントが統合・用語統一・ファイル書き出し
6. ナレッジシェア要約が必要な場合：背景／TAKEAWAY／示唆／論点の4層構造で生成
7. 本ファイル（workflow-record.md）形式で運用記録を残す
