# sales-deck-review

使用タイミング：B2Bセールス資料（特に日本語・代理店提案）のレビューやリライトを求められたとき。

## 優先チェックリスト（上から順に実行）

### 1. コピーの具体度

- タグライン／ヘッドラインが **主語 × 動詞 × 目的** で完結しているか
- 抽象語（「気づき」「体験」「共感」）を定義なしで使っていないか
- **上から目線**（「〜できます」「〜組めます」「代理店が」）を検出 → 「〜いただけます」「代理店様」に言い換え
- 無根拠の最上級（「最も」「業界最大級」「唯一」）→ 実証可能な表現に置換

### 2. ハルシネーション検出

- すべての数値にソースURL + 発行日が紐づいているか
- 派生数値（「80%が…」など）が一次ソースから導けるか
- 競合情報が公式／IR／主要紙ソースか
- 代理店ヒアリング等の**非公開データを資料に載せていない**か

### 3. 競合比較表

- 「非公表 / 要相談」が半数以上の行は **削除**
- 自社が負ける軸は載せない or 軸名を変えて文脈化
- 自社の弱みを **自分で surface しない**（留意点行なし）
- 商品名（MTA / GOAT / AnyTag 等）だけでなく **解釈した強み** を主文に
- 競合カテゴリによって領域を限定（例：総合広告グループなら "インフルエンサー領域での" 強みに限定）

### 4. レイアウト違和感

- **横揃え**：複数カードの proof box / KPI / CTA が横一線にあるか
- **縦揃え**：下段（.srcLine / .caseNote / .footnote）が下端に pin されているか
- **改行**：日本語が文節境界で改行されているか。`word-break: auto-phrase; line-break: strict;` が設定されているか
- **文字切れ**：`transform: scale()` や `bottom: -x` で突き抜けた要素がないか
- **重複情報**：見出しに含まれる数字を直下の大数字で繰り返していないか

### 5. 情報設計

- 1ページ 1メッセージになっているか
- 自社紹介より先に売り込みを始めていないか（P2=会社紹介、P5以降=ポジショニング）
- 各ページの役割（静→知→強→信→実務→決）が通っているか

## 重要原則

1. **敵地で戦わない**。自社が負ける土俵には乗らない
2. **断言 > 婉曲**。曖昧な敬語で戦闘力が落ちる
3. **数字は必ず文脈化**（「25,000名の "稼働" 数」vs 相手の「400万 "累計" DB」）
4. **実名 vs 匿名** の判断：一次公開ソースあれば実名、なければ匿名＋口頭共有

## 出力フォーマット

レビュー時は下記の形で返す：

```
## ❌ 要修正（Critical）
- [ページ/要素]: 問題 → 推奨修正

## ⚠️ 検討（Warning）
- [ページ/要素]: 気になる点 → 代替案

## ✅ 維持（Keep）
- 特筆すべき良い点
```

リライト時は、修正前後の比較を必ず見せる。
