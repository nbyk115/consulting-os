# 戦略判断ログ（佐藤裕介フレーム適用）

戦略決定の根拠と撤退基準を、迷ったときに立ち戻れる形で残す原典。
佐藤裕介フレーム原則：判断は記録しろ、後から自分を甘やかすな。

---

## 判断1：OEM販売をやるか？

問い：このコンサルOSを他社にOEM販売＋ランニング品質改善サポートとして提供すべきか。

結論：条件付きYES（副業路線で）。本業として張るのはNO。

### 判断の根拠

#### NO側の論点（本業として張るには弱い）

| 論点 | 内容 |
|---|---|
| 自社販売の機会損失 | OEMに出すコストの大半は「自社で直販すれば取れたはずの利益」 |
| ランニング = 人月の罠 | 「品質改善サポート」は見た目SaaSだが、実態は人月。スケール曲線が立たない |
| 3年で剥がされる構造 | OEM先は売れたら自社化したがる。LTV最大化が構造的に困難 |
| ブランド毀損リスク | OEM先の品質管理ミスでこちらの本業ブランドが傷つく可能性 |

#### YES側の論点（副業ならハマる）

| 論点 | 内容 |
|---|---|
| キャッシュフロー副業として優秀 | 月額50〜100万 × 2〜3社 = 月100〜300万。本業との時間配分次第で十分成立 |
| 本業のR&D装置になる | OEM先の運用データ・改善要望から、本業に効く改善が抽出できる |
| プロダクト化への踏み台 | 副業で得た知見を、将来SaaS化する際の検証データとして使える（やるかは未定でOK） |
| 紹介経由なら営業コストほぼゼロ | 副業だから営業しない、紹介だけ受ける、で十分回る |

### YES条件の設定

副業として成立させる5つの構造的制約：

1. 顧客は2〜3社まで（時間配分死守）
2. 高単価・低工数（月20時間以内、超過は追加課金）
3. カスタマイズ原則拒否（するなら別料金）
4. ナレッジ資産で殴る（人月で殴らない）
5. 3年で剥がされてOK（LTV執着しない）

→ この5つを契約レベルで担保できないなら、そもそも受けない。

## 判断2：プロダクト化（SaaS化）するか？

問い：OEM先で得たデータ・知見をもとに、コンサルOSをSaaSプロダクト化すべきか。

結論：現時点ではNO（保留）。少なくとも副業フェーズではやらない。

### 判断の根拠

- SaaS化は最低でもエンジニア2〜3人月、年間数百万のインフラ・運用コスト
- 副業フェーズの収益（月100〜300万）では投資回収が苦しい
- プロダクト化に踏み込むと「副業」の定義から外れる＝本業との境界が崩れる
- やるなら「本業として張り直す」決断を別途行う必要がある

### 再検討トリガー

以下のいずれかが満たされたら、SaaS化を再検討する：

- OEM案件が累計5社を超え、共通要望が明確に抽出できている
- 月次収益が500万を安定的に超える
- 本業フェーズの転換期（大型エグジット、事業整理等）

## 判断3：価格レンジをいくらに設定するか？

問い：初期導入と月額の価格レンジをどう決めるか。

結論：初期300〜500万 / 月額50〜100万（Standardプラン400万＋70万を初期提示）

### 判断の根拠

- 下限の根拠：月額50万を下回ると、月20時間の工数（≒7.5万 × 20h = 15万原価）を引いた粗利が薄く、副業として割に合わない
- 上限の根拠：100万を超えると「本業コンサル」と顧客が認識し、SLA・成果保証を求められ始める。副業のフレームを超える
- 初期導入の意義：着手前前払いで未収リスクを排除＋セットアップ工数の確実な回収

### 値引きしないライン

月額の30%以上は絶対に値引かない。それを欲しがる相手はそもそも副業の対象外（→ `03-ideal-partner-profile.md` Red Zone）。

## 判断4：誰を取らない（断る）か？

問い：どんな引き合いを断るか。

結論：以下のいずれか1つでも該当したら即断る。

- 「全部丸投げしたい」「うちは運用できない」
- 24/7サポート・即時対応必須
- 成果保証・KPIコミット要求
- ソースコード／プロンプト開示要求
- 月額50万を「高すぎる」と即答する
- 本業との直接競合
- 反社・コンプラリスク
- 直感で「合わない」

### 判断の根拠

副業の最大の損失は「断れずに受けた合わない案件」。1件の不適切な案件は、3件の適正案件分の精神的・時間的コストを吸う。

→ 断ることを躊躇わない。断りトークは `03-ideal-partner-profile.md` に常備。

## 撤退基準（あらかじめ決めておく）

副業として続ける／畳むの判断基準を、感情論ではなく数値で決めておく。

### 個別契約の解約基準

| トリガー | アクション |
|---|---|
| 月次工数20時間を3ヶ月連続超過 | 値上げ交渉 → 拒否されたら次回更新で解約 |
| 支払い遅延が2回以上 | 契約見直し → 改善なければ次回更新で解約 |
| 担当者との関係悪化 | 1四半期様子見 → 改善なければ次回更新で解約 |
| 本業への明確な阻害 | 即座に解約交渉 |

### 事業全体の撤退基準

以下のいずれかを満たしたら、OEM事業から撤退を真剣に検討：

- 本業年商に対するOEM貢献が10%未満で、かつ月間時間の20%以上を消費
- 直近6ヶ月で「断れずに受けた」案件が2件以上発生
- OEM事業起因のストレスで本業パフォーマンスが落ちている
- コンサルOS本体の進化スピードが、本業ニーズではなくOEM先ニーズで決まり始めている

### 撤退時の動き

1. 新規受注を即停止
2. 既存契約は満了まで履行（途中解約はトラブルの元）
3. 最後の更新時に「次回更新なし」を60日前通告
4. 引き継ぎ先を紹介できれば紹介、できなくてもOK

## 自己点検チェックリスト（四半期ごと）

副業として健全か、四半期ごとに以下を自問する。

- [ ] 同時稼働OEM先は2〜3社の範囲に収まっているか？
- [ ] 月次工数20時間/社の上限は守られているか？
- [ ] 値引きはルール（30%以下）の範囲内か？
- [ ] 「全部任せたい」系の案件を受けていないか？
- [ ] 本業の時間・パフォーマンスに影響が出ていないか？
- [ ] 直近で「断った」案件があるか？（断れていない＝Red Zone）
- [ ] OEM収益が本業収益の補完として機能しているか？

3つ以上「No」なら、戦略の見直し時期。
