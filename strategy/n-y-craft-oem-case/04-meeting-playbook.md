# 打合せ運営の段取り＋想定問答（自分用）

打合せ前にこのファイルを一通り読む。
佐藤裕介フレーム原則：相手の熱量に押されない、価格は最後に出す、断る覚悟を持って臨む。

---

## 1. 打合せのゴール設定

### 必達ゴール

- 提案内容（4層構造）に原則合意を取り付ける
- 価格レンジ（プロジェクト70万・ランニング月12万・成果報酬5%/15%）を呑むかどうかの感触を得る
- 次のアクション（詳細見積・契約書ドラフト）へのコミットメントを取る

### あったら嬉しいゴール

- N.Y.CRAFT の実際の月商感覚を聞き出す
- 関根さんが想定する「使い手」の具体像（社内のスペシャリスト誰か）を確認
- リファレンス事例化への協力に対する温度感を確認

### 守るべきライン（撤退条件）

- 死守価格ライン（プロジェクト70万 / ランニング月10万 / 成果報酬3%）を下回る要求
- 知財所有権の譲渡要求
- 24/7 サポート・成果保証等の SLA 要求
- → これらが出たらその場で受諾せず、持ち帰る

---

## 2. 提示の順序（絶対に守る）

### NG パターン：金額から話す

```
「価格は初期70万、月12万で、成果報酬が5%です」
   ↓
関根さん：「え、結構するね…」（ドン引き）
   ↓
価値の説明をしても後の祭り
```

### 正しい順序

```
ステップ1：ヒアリング（相手の熱量と期待を再確認）
   ↓
ステップ2：価値の言語化（NotebookLMとの違い）
   ↓
ステップ3：構造の説明（4層・役割分担）
   ↓
ステップ4：成果報酬の魅力（同じ船に乗る構造）
   ↓
ステップ5：リファレンス案件としての位置付け
   ↓
ステップ6：金額提示（最後）
   ↓
ステップ7：次アクションのコミット
```

---

## 3. 各ステップの台本

### ステップ1：ヒアリング（10分）

目的：関根さんの期待・温度感の最終確認、月商感覚の聞き出し

質問例：
- 「メッセージ拝見しました。改めて、関根さんver の OS で何を実現したいか、もう少し具体的に教えていただけますか？」
- 「使い手として想定されているスペシャリストの方、社内に何人くらいいらっしゃいますか？」
- 「現状、関根さんが営業資料・トーク作成等にどれくらい時間使われてますか？」
- 「N.Y.CRAFT として、こういったツール／システムへの予算感はどれくらいでお考えですか？」（月商を直接聞かず、予算感で察する）

やってはいけないこと：
- 自分から先に金額や提案内容をぶつけない（相手の口から期待値を引き出す）

### ステップ2：価値の言語化（10分）

目的：NotebookLM では届かなかった理由を構造的に説明する

話し方：

> 「NotebookLM 試されたとのことですが、辞書代わりにしかならなかったお気持ち、よく分かります。
> なぜそうなるかというと、NotebookLM は RAG型の知識検索ツール なんです。
> 文書から関連情報を探して引用するのは得意ですが、判断や思考を再現することはできません。
>
> 関根さんが欲しいのは『関根さんならどう判断するか』を返してくれる 判断装置（クローン） ですよね。
> これは設計思想が全く違うアプローチが必要で、コンサルOS はまさにそこを実装するためのシステムなんです」

ホワイトボード or 紙に書く図：

```
NotebookLM      →  辞書（何が書いてあるかを返す）
コンサルOS      →  判断装置（関根さんならどう判断するかを返す）

  ↑
  この差は「ソース足せば改善する」じゃなくて
  「設計思想が違う」のでアプローチを変える必要がある
```

### ステップ3：構造の説明（15分）

目的：4層構造と役割分担を理解してもらう

話す順序：
1. 役割分担の図（関根さんがフロント、自分がバックエンド）を提示
2. 4層（業務委託・プロジェクト・ランニング・成果報酬）の意義を説明
3. それぞれが「何への対価か」が違うので分けている、と伝える

重要なメッセージ：

> 「関根さんは顧客との関係性に集中していただいて、こちらは OS の品質維持に集中する。
> その分担を契約レベルで明確にしておきたいんです」

### ステップ4：成果報酬の魅力（10分）

目的：「同じ船に乗る」構造で関根さんを安心させる

話し方：

> 「成果報酬を組み込むのは、こちらが N.Y.CRAFT さんと 同じ船に乗る という意思表示でもあります。
> ベースの料金だけだと、こちらは導入が決まれば終わりですが、
> 成果報酬があると N.Y.CRAFT さんが伸びれば伸びるほどこちらも伸びる。
> なので継続的な改善・サポートのインセンティブが構造的に組み込まれます」

論点になる場合の補足：
- 対象は契約後の新規受注のみ（既存顧客は対象外）
- 期間は24ヶ月で打ち切り（永続ではない）
- 関根さんの自発的な営業努力が主で、OSは補助なので料率は5%（控えめ）

### ステップ5：リファレンス案件としての位置付け（5分）

目的：価格圧縮の正当化＋関根さんへのメリット提示

話し方：

> 「実は今回、関根さんとの案件を コンサルOSのOEM展開における第1事例（リファレンスケース）
> として位置付けたいと考えていまして。
>
> プロジェクトフィーを通常より圧縮させていただく代わりに、
> N.Y.CRAFT さんに先行ユーザーとしての発信協力をお願いできればと思っています。
>
> 関根さんにとっても、コンサルOSの先行事例として登壇・記事等の発信機会につながります」

→ ここで関根さんが「面白そう！」となれば、価格交渉の地ならし完了。

### ステップ6：金額提示（10分）

目的：価格と内容のセットで提示する

話し方：

> 「価格の構造は4層になっていまして：
>
> ① 既存の業務委託は月5万のまま、据え置き
> ② プロジェクトフィー（OS構築）が着手時一括で70万円
> ③ ランニング（月次品質サポート）が月12万円
> ④ 成果報酬は通常受注で5%、OS自体の再販時は15%（24ヶ月）
>
> 通常はプロジェクトが120万くらいなんですが、リファレンス案件として70万に圧縮しています」

ポイント：
- 通常価格（120万）を一度言ってから割引価格（70万）を提示する → アンカリング効果
- 4層をセットで見せる → 単一料金より「何にいくら」が分かりやすい
- 一気に全部出す → 後出しジャンケンに見えない

### ステップ7：次アクションのコミット（10分）

目的：「いいね」で終わらせず、具体的な次の一歩を決める

話し方：

> 「ご質問・ご懸念点いかがですか？
> 大筋でご検討いただけそうでしたら、こちらで詳細見積書と契約書ドラフトを2週間以内に作成します。
> その後改めて打合せして、合意できれば構築フェーズに入る、という流れでいかがでしょう」

取り付けたいコミット：
- 見積書受領後の検討期間（目安2週間以内）
- 契約締結希望時期
- Phase 1 着手希望時期

→ ここを曖昧にすると「考えます」のまま2ヶ月音沙汰なし、になる。

## 4. 想定質問と返答

### 価格に関する質問

| 質問 | 返答 |
|---|---|
| 「もう少し安くならない？」 | 「すでに通常120万のところを、リファレンス案件として70万まで圧縮しています。これ以上は工数的に難しいですが、Phase 1 のスコープ調整は可能です」 |
| 「最初は無料で試させて」 | 「無料は難しいですが、最初の1〜2ヶ月のみランニングを月8万に圧縮するお試し設計はご相談可能です」 |
| 「成果報酬は無しでいい？」 | 「その場合、ベース料金を上げる必要があります。プロジェクト100万・ランニング月15万にしてはいかがでしょう」 |
| 「全部買い取りたい」 | 「買取の場合は800〜1000万のレンジになります。年間契約の数年分相当をいただく形です」（実質お断り） |

### スコープに関する質問

| 質問 | 返答 |
|---|---|
| 「業界特化テンプレも作って」 | 「Phase 2 で対応します。1業界あたり50万円〜のレンジでお見積します」 |
| 「うちの社員10人全員に研修して」 | 「キックオフ研修は90分含まれます。それ以上は1回30万でオプション提供します」 |
| 「24時間サポートして」 | 「副業として運営しているため、24/7サポートは構造的に提供できません。営業日48h以内応答が上限です」 |
| 「成果保証してほしい」 | 「成果保証は難しいです。代わりに成果報酬を組み込むことで、こちらも継続改善のインセンティブを持つ設計にしています」 |

### 知財・契約に関する質問

| 質問 | 返答 |
|---|---|
| 「ソースコード見せて／開示して」 | 「OS本体の知財はこちら帰属で、ライセンス利用の形になります。これはコアバリューを守るために譲れない条件です」 |
| 「他社にも同じものを提供する？」 | 「コンサルOS本体は他社にもOEM提供しますが、関根さんver のカスタマイズ部分（関根さんの判断ログ・テンプレ）は完全に個別です」 |
| 「契約途中で解約したい」 | 「契約期間（1年）は基本履行をお願いします。事業廃止等の事由があれば違約金（残月額の50%）で途中解約は可能です」 |

### 業務委託との関係

| 質問 | 返答 |
|---|---|
| 「業務委託の延長でやってよ」 | 「業務委託は『関根さん本人へのアドバイス』、今回のOEMは『N.Y.CRAFT という会社への資産提供』なので、対価の対象が違います。別契約として整理させてください」 |
| 「業務委託料を上げて、その中でOEMもやって」 | 「混ぜると後で整理できなくなるので、別建てでお願いしたいです。業務委託の見直しは別途相談しましょう」 |

## 5. NG対応リスト（絶対やらない）

| NG行動 | なぜダメか |
|---|---|
| 関根さんの熱量に押されてその場で値引きする | 副業フレーム破綻 |
| 「友達価格で」と言われて呑む | 業界全体の価格相場を毀損、他案件にも影響 |
| 知財所有権の譲渡 or 共有を呑む | ビジネス終了 |
| 「全部やります」と安請け合い | スコープ膨張で副業破綻 |
| 24/7サポート・即時対応をその場で約束 | 副業として運営不可能 |
| 業務委託フィーにOEM料金を吸収させる | 取り分が大幅に減る |
| 契約書なしで構築開始 | 揉めた時に何も守れない |
| 着手前前払いを呑まずに着手 | 未収リスクを背負う |

→ 上記が出たら 「持ち帰って検討させてください」 で一旦止める。即答しない。

## 6. 打合せ後のアクション

### 帰宅後すぐ（24時間以内）

1. 打合せメモを書く（関根さんの反応、引き出した情報、合意したこと）
2. 約束した次アクション（見積書作成等）の期限を再確認
3. 価格交渉が発生した場合、`03-pricing-rationale.md` の防御線で再評価

### 1週間以内

1. 詳細見積書のドラフト作成
2. 契約書のドラフト作成（弁護士確認は契約直前で OK）
3. Phase 1 構築計画の詳細化（インタビュー日程候補等）

### 2週間以内

1. 関根さんに見積書・契約書ドラフトを送付
2. 2回目打合せ日程の調整

## 7. 打合せ前の自分への問いかけ

打合せ直前に以下を自問する：

- [ ] 副業として受ける覚悟はできているか？
- [ ] 死守ライン（価格・スコープ・知財）を即座に思い出せるか？
- [ ] 「断る」選択肢を持っているか？（呑めない条件なら持ち帰る勇気）
- [ ] 関根さんの熱量に流されない冷静さがあるか？
- [ ] 業務委託の関係を維持しつつOEMだけ断る選択肢を意識できているか？

→ 全て Yes になっていなかったら、打合せの場で慎重に。
