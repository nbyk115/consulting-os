# monthly-impact-report

月次成果レポートの汎用スキル。**売上に直結する指標だけ**を1ページに収める。

## 前提
- `client-context-load` 済み（KPI と目標が定義されている）
- 直近3ヶ月の数値が取得できる状態

## 含めるべき指標（売上直結のみ）

- 新規リード数 / 月（チャネル別）
- リード→商談化率
- 商談→受注率
- 受注金額 / ARPA / LTV
- 解約数（既存ビジネスがある場合）
- NPS / レビュー獲得速度

## 含めない指標（バニティメトリクス）

- 総トラフィック単独
- インプレッション単独
- フォロワー数単独
- ドメイン評価のみ

これらは**売上を生んでいるかを示さない**。先月との比較で文脈なく「成長」を主張するためだけに使われがち。

## 実行手順

1. クライアント KPI を `client-context.md` から取得
2. 直近3ヶ月の数値を取得（クライアント提供 or ツール連携）
3. 前月比 / 前年同月比を算出
4. **「先月効いた施策／効かなかった施策」を1行ずつ**特定
5. 来月の集中投下対象を1〜3個に絞る

## 出力フォーマット（1ページ厳守）

```markdown
# [クライアント名] Monthly Impact Report — YYYY-MM

## サマリー（3行）
- 売上影響:
- 効いた施策:
- 来月の集中投下:

## 主要指標
| 指標 | 当月 | 前月 | 前年同月 | コメント |
|---|---|---|---|---|

## 効いた施策 / 効かなかった施策
- ✅ 効いた:
- ❌ 効かなかった:

## 来月の意思決定リスト（クライアント判断）
- [ ] 集中投下: ...
- [ ] 撤退検討: ...
- [ ] 様子見: ...
```

## アンチパターン

- 指標を10個以上載せる → 1ページに収まらない時点で意思決定ツールにならない
- 「先月比 +5%」とだけ書く → 何のせいで上がったか不明
- AIが「来月の戦略」を結論する → 判断は人間に残す
