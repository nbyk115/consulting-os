## 2. ConsultingOS 自体の ICP（参考デフォルト）

### Primary ICP: マルチエージェント設計を要する事業会社・コンサル・代理店

**属性**
- 規模: 従業員 50-500 名 / 売上 5-100 億円
- 業界: 広告代理店 / コンサルティングファーム / SaaS 事業会社 / D2C ブランド
- 役割: 経営層・戦略責任者・マーケティング責任者・プロダクト責任者
- 地域: 日本国内中心（グローバル展開意欲あり）

**痛み（Pain）**
- 提案書 / 戦略文書のクオリティをスケールさせられない（属人化）
- AI を使いたいが、単発プロンプトでは品質が安定しない
- 複数部門（戦略 / 開発 / クリエイティブ / マーケ）の連携が分断されている
- 経営判断と実装の間に翻訳レイヤーが必要
- 機密性の高いクライアント情報を Cloud LLM に丸投げできない

**ジョブ（Job to be Done）**
- 提案書・戦略分析・LP・セールス資料を **属人化せず量産**したい
- AI を「**規律ある専門家チーム**」として使いたい
- クライアント案件ごとに**ブランドと文脈を維持**したい
- 経営判断（PL・市場構造）から実装（コード・デザイン）まで**一気通貫**で動かしたい

**評価軸（Buying Criteria）**
- 設計の網羅性（経営〜実装の統合度）
- セキュリティ多層防御（Layer 0/1/2）
- 反証モードによる品質ガード
- 実証された参照実装の有無
- カスタマイズ容易性

## 3. 非ターゲット（明示的に外す）

> 佐藤裕介流: 「**敵地で戦わない**」。以下は ConsultingOS の差別化が効かない領域。

- **個人開発者の単発ツール用途**: Cursor / Cline / GitHub Copilot のような IDE 拡張で足りる
- **エンタープライズの完全カスタム要件**: 独自セキュリティ要件 / オンプレ専用環境
- **完全初心者**: Claude Code 自体の基礎が必要（教科書系記事の対象）
- **AI 倫理・コンプライアンスが極端に厳しい業界**: 防衛・医療など、独自法規制対応が必要な領域

## 4. ペルソナ別利用文脈

### Persona A: 戦略コンサルタント（PM / シニア）
- **使用シーン**: 提案書執筆 / 競合分析 / クライアント PL 試算
- **起点エージェント**: `strategy-lead` / `proposal-writer` / `competitive-analyst`
- **重視するもの**: 反証モード、引用ソースの厳密さ、`first-principles-breakdown`
- **避けたいもの**: ハルシネーション、空疎な提案

### Persona B: マーケティング責任者（事業会社 CMO / 代理店プランナー）
- **使用シーン**: チャネル戦略 / SEO / 広告運用 / GEO 設計
- **起点エージェント**: `marketing-director` / `seo-specialist` / `agentic-content`（GEO）
- **重視するもの**: 実データ接続（toprank, /last30days）、ROI 数値化
- **避けたいもの**: 表面的なナレッジ提案、データ無根拠の推奨

### Persona C: クリエイティブディレクター（広告代理店 / D2C ブランド）
- **使用シーン**: LP 制作 / セールス資料 / SNS コンテンツ / 動画
- **起点エージェント**: `creative-director` / `sales-deck-designer` / `frontend-dev`
- **重視するもの**: ブランド一貫性、`DESIGN.md`、Hyperframes での動画量産
- **避けたいもの**: パクリ系ツール（ai-website-cloner 等は禁止）

### Persona D: テック / プロダクト責任者
- **使用シーン**: アーキテクチャ設計 / コードレビュー / AI 機能統合
- **起点エージェント**: `tech-lead` / `fullstack-dev` / `ai-engineer`
- **重視するもの**: セキュリティ Layer 0/1/2、反証モード、`engineering-playbook`
- **避けたいもの**: 単発のコード生成（規律のない出力）

## 5. 価値提案（Why ConsultingOS）

| 競合カテゴリ | 競合の強み | ConsultingOS の差別化 |
|---|---|---|
| 単発プロンプト集 | 即使える | 反証モード + 自己進化 + 多層防御で**長期運用品質**が桁違い |
| 個別 AI ツール（Cursor / v0 等） | 領域特化で深い | **6部門統合**による経営判断 → 実装の一貫性 |
| 大手コンサルファーム | 人的リソース | **再現可能な構造化された手法**、属人化排除 |
| 自前 LLM 構築 | 機密対応 | **すぐ動く** + Claude Code 公式エコシステム + Layer 0/1/2 |

**コア価値**: 「**経営判断統合支援を AI 駆動で再現可能にする唯一級の OS**」
