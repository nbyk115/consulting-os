# サービス定義書

提供するもの／提供しないものを明確化することで、副業として運用負荷を制御するための原典。
カスタマイズ依頼が来たときの「原則お断り」の根拠として、契約書に添付する。

---

## 1. 提供するサービス（Inclusions）

### 1.1 初期導入パッケージ（買い切り）

| 項目 | 内容 |
|---|---|
| OS本体提供 | コンサルOS（Claude Code 構成一式 + プロンプト/スキル/エージェント定義） |
| 構築支援 | OEM先環境への導入セットアップ（リモート2回／合計4時間） |
| 運用ガイド | 運用手順書・ベストプラクティス集 |
| キックオフ研修 | 90分 × 1回（Zoom録画あり、社内共有可） |
| 初期テンプレート | 基本5テンプレート（リサーチ／翻訳／要約／インタビュー整理／提案資料） |

完了定義：OEM先の指定担当者が、提供テンプレートを使って単独で1案件分の成果物を出せる状態。

### 1.2 ランニング品質改善サポート（月額）

| 項目 | 内容 | 上限 |
|---|---|---|
| 月次レビュー会 | 直近案件の品質レビュー＋改善提案 | 90分 × 月1回 |
| ベンチマーク提供 | 業界水準・他事例（匿名化）との比較診断 | 月1回 |
| テンプレ更新 | 既存テンプレの改訂版配布 | 四半期1回 |
| Slack/メール質問対応 | 運用上の判断相談 | 営業日48h以内返信 |
| 総工数上限 | 1社あたり | 月20時間 |

月20時間を超過した場合：超過分は人月単価（月額の1/20）で追加請求 or 翌月相殺。
3ヶ月連続超過でプラン見直し協議（値上げ or 解約）。

## 2. 提供しないもの（Exclusions）

明示的に契約書から除外する項目。引き合い時に「これも込みで」と言われたら断るか有償オプション化する。

### 2.1 個社カスタマイズ

- OEM先固有のテンプレート新規開発
- OEM先固有のエージェント設計
- OEM先固有のスキル定義
- 既存テンプレートの大幅改修（軽微な調整は含む）

→ やるなら別契約。人月単価で見積（目安：120万/人月）。

### 2.2 オペレーション代行

- OEM先案件の実作業（リサーチ・翻訳・資料作成）
- OEM先のクライアント対応
- OEM先従業員の教育・トレーニング（キックオフ研修90分以上）

→ これは本業のコンサル業務になる。副業ラインでは絶対に受けない。

### 2.3 24/7サポート・SLA保証

- 即時対応・夜間/休日サポート
- 稼働率SLA（99.9%等）
- ペナルティ条項付き応答時間保証

→ 副業の前提を破壊する。受けない。

### 2.4 知財の譲渡・独占ライセンス

- コンサルOSのソースコード／プロンプト／スキル定義の譲渡
- OEM先による再配布権
- 業界・地域独占ライセンス（例外あり、後述）

→ 譲渡は不可。独占ライセンスは「3年以上の前払い」かつ「年額の3倍以上」なら検討余地あり（佐藤裕介フレーム：イグジット価格を先に決める）。

## 3. スコープ境界（よくあるグレーゾーンの判断）

| 引き合い内容 | 判断 | 根拠 |
|---|---|---|
| 「テンプレを1つ追加してほしい」 | 別料金（30〜50万） | 個社カスタマイズに該当 |
| 「自社ロゴ入りのアウトプットにしたい」 | OK（無償、テンプレ側に変数化済み） | 軽微 |
| 「他社事例を匿名で見せてほしい」 | OK（ベンチマーク提供の一環） | 含まれる |
| 「うちの業界向けにOSを再設計したい」 | 別契約（数百万〜） | 大幅改修 |
| 「成果物の品質保証をしてほしい」 | 不可 | オペ代行になる |
| 「うちの社員に毎週レクチャーしてほしい」 | 不可（キックオフ以外） | オペ代行になる |
| 「Slackチャンネル常駐してほしい」 | 不可（メール/Slack 48h応答が上限） | SLA超過 |

---

## 4. 提供品質の定義（Definition of Done）

「品質改善サポート」が抽象的なまま受けると無限に消耗するため、月次レビューの完了定義を明文化する。

月次レビューの完了条件：

1. OEM先から事前に提出された「直近の成果物1〜3点」を確認
2. レビューシート（提供テンプレ）に沿って指摘・改善案を記載
3. 90分のレビュー会で口頭フィードバック
4. レビューシートをPDFで納品

→ これ以上は「別件」。Slackで追加質問が来ても、レビューシートに記載済みなら「シート参照」で返す。
