## 1. この案件をなぜ受けるのか（戦略的意義）

副業のキャッシュフローだけで見れば、月5万関係の相手からOEMを取りに行くのは効率が悪い。
それでも受ける意義は 3つ：

### ① OEM事業の第1リファレンスとして

- コンサルOSのOEM展開は事例ゼロからのスタート
- N.Y.CRAFT は SMB／レガシービジネス領域 = コンサルOSが効きやすい標的領域
- 関根さん（経営者）と直接組める = 意思決定が速い、運用フィードバックが純粋
- → ここで成功事例を作れれば、その後のOEM営業の難易度が大幅に下がる

### ② 自分の本業の周辺ネットワーク強化

- 関根さんは中小企業の経営層との接点が深い
- リファレンス案件を成功させることで、関根さん経由で他のSMB／関連事業者との接点が広がる
- 副業の収益単体では薄くても、ネットワーク資産としてのリターンが乗る

### ③ コンサルOS自体のR&D

- 関根さんは NotebookLM で挫折済み = 「思考の再現」というユースケースの最先端ニーズを持つ顧客
- ここで作る関根さんver は、コンサルOS の「人格化／クローン化」機能の最初の実装
- → 他のOEM先にも横展開できるアセットになる

## 2. 4つの収益層構造の設計思想

### なぜ層を分けるのか

> 対価の対象が違うものを一つの料金に丸める = 後で揉める根本原因

| 層 | 対価の対象 |
|---|---|
| ①業務委託（月5万） | 関根さん本人へのアドバイス労働（時間） |
| ②プロジェクトフィー（70万） | 資産（OS）の構築（一回もの） |
| ③ランニング（月12万） | 資産の品質維持・運用サポート（継続労働＋ナレッジ提供） |
| ④成果報酬（5% / 15%） | OS活用による事業上のアップサイド（成果連動） |

→ それぞれ性格が違うので、合算せず分けて契約することで透明性が上がる。

### 4層の意義（佐藤裕介フレーム）

- ①既存維持：信頼関係の継続。OEM化で吸収させない
- ②前金で原価回収：構築工数を着手前に確保 → 未収リスクゼロ
- ③ベースキャッシュ：副業として最低限のキャッシュフローを保証
- ④アップサイド：N.Y.CRAFT が伸びれば自分も伸びる構造（営業ゼロでスケール）

## 3. 「リファレンス案件」としての価格設計

### 70万にする戦略的根拠

通常レンジ（100〜150万）から30〜80万を割引く。これを「ただの値引き」にしない設計：

| 自分が払う対価 | 関根さんから受け取る対価 |
|---|---|
| プロジェクトフィー 30万分 | N.Y.CRAFT 名出しでの事例公開権 |
| | 構築プロセス・成果のケーススタディ化権 |
| | 他社OEM営業時のリファレンスコール協力（年2回まで） |
| | OS横展開時のナレッジ蓄積（関根さんver で得た知見の再利用権） |

→ 30万 = 「リファレンス取得への投資」。値引きではなく等価交換。
→ 関根さんも「タダで安くしてもらった」感がなく、対等な関係で進められる。

### Phase 分割の意義

70万 = Phase 1（コア機能のみ）に限定。これにより：
- 構築スコープが明確化 → 工数膨張を防ぐ
- 関根さんも「最初の70万で効果を見極めてから判断できる」のでリスクが下がる
- 効果が見えたら Phase 2（追加50〜80万）を実績ベースで売れる

## 4. 知財・所有権の防衛線

### 絶対譲らないライン

- コンサルOS本体の著作権・知的財産権は全て自分側に帰属
- 関根さんver 化された部分も自分帰属
- N.Y.CRAFT はライセンス利用のみ、再販は別契約必須

### 「全部買い取りたい」と言われた場合の防御

関根さんが OS を買い取りたいと言ってきた場合の提示価格：
- 年額契約の5倍以上 + 一括前払い
- 例：年額 144万 × 5 = 720万＋プロジェクトフィー相当 = 800〜1000万
- 実質お断り価格として機能する設計

この水準を呑むなら、こちら側は本業として張り直す覚悟ができる金額。
呑まないなら、ライセンスモデルが守られる。

## 5. 撤退基準（先に決めておく）

副業として続ける／畳むの判断を、感情論ではなく数値で決めておく。

### 個別契約の解約基準

| トリガー | アクション |
|---|---|
| 月次工数 8時間を3ヶ月連続超過 | 値上げ交渉 → 拒否されたら次回更新で解約 |
| 支払い遅延が2回以上 | 契約見直し → 改善なければ次回更新で解約 |
| 関根さんとの関係悪化（業務委託も含む） | 1四半期様子見 → 改善なければ全契約解約 |
| 本業（アドテクコンサル等）への明確な阻害 | 即座に解約交渉 |
| 成果報酬の対象売上で揉める | 1度なら協議、再発したら解約 |

### 事業全体の撤退基準（OEM事業として）

以下のいずれかを満たしたら、OEM事業全体から撤退を真剣に検討：

- N.Y.CRAFT 案件が初年度赤字（時給換算1万円以下）で、Phase 2 拡張も見えない
- 半年以内に2件目のOEM案件が見えてこない（ネットワーク効果が出ない）
- リファレンス事例化が機能しない（関根さんが事例公開を渋る等）
- コンサルOS本体の進化が、関根さんver ニーズに引きずられて他案件に効かなくなる

## 6. 判断ログ：佐藤裕介フレームでの主要判断

### 判断1：受けるか？

結論：受ける（条件付き）

- NO論：月5万関係の相手から本格OEM取るのは飛距離が大きい、関係性破壊リスク
- YES論：第1リファレンス案件としての価値、関根さんから既にニーズ顕在化、信頼関係あり
- → 第1リファレンスとしての価値が、価格抑制の機会損失を上回る と判断
- 条件：プロジェクトフィー＋ランニング＋成果報酬の3層を別契約で締結、業務委託は分離

### 判断2：価格レンジ

結論：プロジェクト70万 / ランニング月12万 / 成果報酬5% (再販15%)

- N.Y.CRAFT の月商規模（推定80〜400万）から逆算：ランニングは月商の5〜10%以内に収める
- 既存業務委託 月5万からの心理的距離：ランニング月12万 = 業務委託の2.4倍は呑める範囲
- 副業時給：年間収益350万 / 工数140h ≒ 時給2.5万 → 副業の最低ライン死守
- 成果報酬5%：通常コンサル受注で10%は取りすぎ。営業力主役・OS補助役なので5%が業界相場
- 成果報酬15%（再販）：OSが商品そのものなので高め設定OK

### 判断3：プロダクト化（SaaS化）するか？

結論：現時点ではNO（保留）

- 副業フェーズではSaaS化への投資（年数百万）は回収困難
- 関根さんver の知見は蓄積するが、プロダクト化は本業として張り直す決断が必要
- 再検討トリガー：OEM案件が累計5件超 + 共通要望が明確 + 月次収益500万安定超
