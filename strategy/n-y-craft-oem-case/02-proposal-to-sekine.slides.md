# 2. なぜ NotebookLM では届かなかったのか

---

# NotebookLM の本質：RAG型の知識検索ツール

| NotebookLM が得意なこと | 関根さんが求めること |
|---|---|
| 文書から関連情報を**検索・引用** | 関根さんの**思考・判断**を再現 |
| 「何が書いてあるか」を返す | 「関根さんならどう判断するか」を返す |
| 静的な知識ベース | 動的な意思決定エンジン |

### 結論

NotebookLM は<span class="highlight">「辞書」</span>、<br>関根さんが欲しいのは<span class="accent">「判断装置（クローン）」</span>。

→ 設計思想が違うので、ソースを足しても辞書のまま。

---

# 3. コンサルOS が解決できる理由

---

# コンサルOS の本質：判断・思考の構造化システム

### 関根さんver を実現する3つの構成要素

| 要素 | 内容 |
|---|---|
| **① スキル定義** | 関根さんの判断軸・思考フレーム・価値観を構造化<br>例：顧客タイプ別の営業アプローチ判断ロジック |
| **② サブエージェント構成** | スペシャリスト思考を関根さんver で実装<br>例：「LP作成」「営業資料」エージェントが関根さんの価値観でアウトプット |
| **③ 判断ログ（ナレッジ）** | 関根さんの過去判断・成功事例・失敗事例を蓄積<br>クローンが参照することで判断精度が上がる |

---

# NotebookLM との決定的な違い

| 機能 | NotebookLM | コンサルOS（関根さんver） |
|---|:---:|:---:|
| 知識検索 | ◯ | ◯ |
| **判断フレームの再現** | ✗ | <span class="accent">◎</span> |
| **役割別エージェント** | ✗ | <span class="accent">◎</span> |
| **アウトプット品質保証** | ✗ | <span class="accent">◎</span>（バックエンドで品質チェック） |
| 継続的アップデート | △ | ◎ |
| 関根さんver化 | ✗ | ◎ |

# 4. ご提案するパートナーシップの形

---

# 役割分担：フロントとバックエンドの分離

```
┌─────────────────────────────────┐
│  N.Y.CRAFT / 関根さん（フロント）       │
│   ・顧客接点・営業・最終判断              │
│   ・関根さんver OS を活用してアウトプット   │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  関根さんver コンサルOS（中核）          │
│   ・判断装置として機能                  │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  提供者（バックエンド）                  │
│   ・品質保証・継続改善・アップデート      │
└─────────────────────────────────┘
```

→ <span class="highlight">関根さんは関係性に集中、こちらは OS の品質に集中</span>

---

# 提供する 4つの層

| 層 | 内容 | 性格 |
|---|---|---|
| ① 既存業務委託 | 関根さんへの壁打ち＋アドテク領域コンサル | **現状継続** |
| ② プロジェクト | 関根さんver コンサルOS の構築 | **一回もの** |
| ③ ランニング | OS の品質維持・運用サポート・アップデート | **月額** |
| ④ 成果報酬 | OS活用による事業上のアップサイドへの連動 | **売上連動** |

→ 対価の対象が違うものを分けて契約することで透明性が上がる

| ⑥ キックオフ研修 | 1日 | スペシャリスト向け使い方研修（90分） |

# Phase 1 のゴール（完了定義）

### 達成状態

関根さんが手を動かさなくても、スペシャリストが<br>関根さんver OS を使って**基本5種類のアウトプットを単独で出せる**状態

関根さんがレビューする際、<br>「これは自分の判断軸と一致している」と感じられる精度

### Phase 1 で含まれないもの（Phase 2 / 別料金）

- 業界特化テンプレ
- 高度なサブエージェント構成（複数連携）
- 関根さん以外のスペシャリスト向け個別カスタマイズ
- N.Y.CRAFT 顧客への OS 再販・OEM 提供（→ 別契約）
