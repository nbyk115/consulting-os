# 理想OEM先プロファイル（受ける／断る基準）

副業として運用負荷を最小化するため、**「誰と組むか」を「何を売るか」より厳しく選ぶ**。
引き合いが来た瞬間に判断するためのチェックリスト。

**佐藤裕介フレーム原則**：合わない相手と組むコストは、売上の機会損失より遥かに大きい。

---

## 1. 受けるOEM先のプロファイル（Green Zone）

以下を**3つ以上満たすこと**を最低条件とする。

### A. 自走できる組織であること

- [ ] 社内に**運用担当者を最低1名アサインできる**（兼任でも可、ただし週8時間以上は確保）
- [ ] その担当者がドキュメント読解・PCツール使用に抵抗がない
- [ ] 経営層／事業責任者が「自分達で使いこなす」コミットメントを持っている

→ 「全部任せたい」「うちの社員には難しい」と言う相手は地雷。

### B. 自社販売チャネルを持っていること

- [ ] OEM先が**既存顧客資産・販売網**を持っている（営業組織、業界ネットワーク等）
- [ ] こちらが直販で届かない領域（業界特化／地域特化／既存契約経由）にアクセスできる
- [ ] つまり「OEM先がいなければ届かない顧客」が存在する

→ ここが弱いと「自社で直販した方が儲かる」状態になり、OEMの存在意義が消える。

### C. 適正な価格感を持っていること

- [ ] 月額50〜100万を「妥当」と即答できる規模感／予算感
- [ ] 値引き交渉の幅が30%以内に収まる
- [ ] 「もっと安くなれば検討」と最初から値段で攻めてこない

→ 安く買い叩こうとする相手は、運用フェーズでも工数を要求してくる傾向が強い。

### D. 契約・支払いの健全性

- [ ] 上場企業 or 売上10億以上の中堅企業（与信OK）
- [ ] 前払いに応じられる
- [ ] 経理・法務部門が機能している（契約交渉が長期化しない）

### E. ビジネス相性

- [ ] こちらの本業と**直接競合しない領域**で使う
- [ ] こちらの本業のブランドを毀損しない事業を運営している
- [ ] 担当者と話していてストレスがない（直感重要）

## 2. 断るOEM先のプロファイル（Red Zone）

**1つでも該当したら即断る**。例外なし。

| 該当項目 | なぜ断るか |
|---|---|
| 「全部丸投げしたい」「うちは運用できない」 | オペ代行に引きずり込まれる |
| 「24/7サポート必須」「即時対応必須」 | 副業の前提崩壊 |
| 「成果保証してほしい」「KPIにコミットして」 | OEM販売の枠を超えてリスクを負う構造 |
| 「ソースコード／プロンプトを開示してほしい」 | 知財コアを渡したらビジネス終了 |
| 「業界独占ライセンスを安く」 | 適正対価（年額×3倍以上、3年前払い）以下なら絶対NO |
| 月額50万を「高すぎる」と言う | 価格感が合わない＝運用フェーズで揉める |
| こちらの本業と直接競合する | 利益相反、本業を傷つける |
| 担当者がコロコロ変わる／意思決定者が見えない | 運用フェーズで大事故になる |
| 反社チェック・コンプラチェックで引っかかる | 言うまでもない |
| 直感で「合わない」と感じる | 副業は精神衛生が最重要KPI |

---

## 3. 紹介経路の優先順位

副業ゆえに**営業活動はしない**。引き合いの入り口を絞る。

| 経路 | 対応 |
|---|---|
| 既存OEM先からの紹介 | **最優先**で検討（紹介者経由の信頼性） |
| 本業の取引先・同業者からの紹介 | 検討（ただし本業との利益相反チェック必須） |
| 知人経由の紹介 | 検討（プロファイルチェック厳格に） |
| Web経由の問い合わせ | **原則受けない**（フィルタ薄い、価格交渉になる） |
| コールドメール／飛び込み | **受けない** |
| 広告／マーケ施策 | **やらない**（副業の前提崩壊） |

## 4. 引き合い時のスクリーニング会話テンプレ

初回ヒアリング（60分）の冒頭15分で以下を確認。Red Zoneに該当したら丁寧にお断りする。

### 確認すべき5つの質問

1. **「導入後、御社内で運用を担当される方はどなたですか？週何時間くらい確保できそうですか？」**
   → 自走できるかの確認

2. **「このOSを使って、どんな顧客／どんな案件で価値を出したいとお考えですか？」**
   → 自社販売チャネル＆本業との競合チェック

3. **「予算感としてはどのくらいをイメージされていますか？」**
   → 価格感のすり合わせ（こちらから先に料金を出さない）

4. **「導入の意思決定はどなたがされますか？経理・法務の体制は整っていますか？」**
   → 与信＆契約進行の現実性確認

5. **「期待される成果やKPIがあれば教えてください」**
   → 「成果保証」を求めてないかの確認

### お断りトーク（汎用テンプレ）

```
ご相談ありがとうございます。お話を伺って、私どもがご提供できる
範囲と、御社が求められている水準にギャップがあると感じています。

特に [運用体制／価格感／成果保証／カスタマイズ範囲] の点で、
私どもの標準的なご提供形態では十分なご支援が難しそうです。

別の形でご紹介できる先があれば改めてご連絡しますが、
今回はお見送りさせていただければと思います。
```

→ **断ることを躊躇わない**。副業の最大の敵は「断れずに受けた合わない案件」。

## 5. 同時稼働数の上限

| 状態 | OEM先数 | 対応 |
|---|---:|---|
| 通常運用 | **2〜3社** | 引き合いがあっても新規はウェイティング |
| 本業繁忙期 | **1〜2社** | 既存先のみ。新規は受けない |
| 本業大型案件中 | **0〜1社** | 既存契約も解約／一時停止を検討 |

**上限を破って受けない**。「もう1社くらいなら」が副業破綻の典型パターン。

## 6. 既存OEM先の年次レビュー

毎年契約更新の前に以下を自己評価し、継続／値上げ／解約を判断する。

| 評価項目 | 継続 | 要見直し | 解約検討 |
|---|---|---|---|
| 月次工数 | 上限内 | 8割以上 | 連続超過 |
| 担当者との関係 | 良好 | 微妙 | ストレス源 |
| 支払い | 期日通り | 遅延あり | 督促必要 |
| 本業への影響 | なし | 軽微 | 阻害している |
| 紹介発生 | あり | なし | ネガティブ評判 |

「要見直し」が3つ以上 → **値上げ交渉**（断られたら解約）
「解約検討」が1つでも → **更新しない**

## 7. 撤退の合図（事業全体としての撤退判断）

副業全体として、以下のいずれかを満たしたら**OEM事業から撤退**を真剣に検討する：

- 本業の年商に対してOEM事業の貢献が10%未満で、かつ自分の月間時間の20%以上を消費している
- 直近6ヶ月で「断れずに受けた」案件が2件以上発生
- OEM事業起因のストレスで本業のパフォーマンスが落ちている
- コンサルOS本体の進化スピードが、本業ニーズではなくOEM先ニーズで決まり始めている

**撤退時の動き**：既存契約は満了まで履行、新規受注を停止、最後の更新時に「次回更新なし」を通告。
